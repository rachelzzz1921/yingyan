# 评测方法论与诚实边界(本轮"真实"怎么兑现)

> 本文件记录**事实性**的设置(环境、模型选型、口径),数字结论见 `report.md` 与 `report_tables_v6.md`(均由脚本从真跑结果生成,绝不手填)。

## 1. 环境与"真实"的兑现

- **能联网**:✅ 已验证(api.anthropic.com 可达、api.minimaxi.com 真实返回)。
- **真实模型调用**:✅ 全程真调 MiniMax(OpenAI 兼容端点 `chatcompletion_v2`),零模拟、零写死示例。每次调用的原始输出落盘 `results/raw/<hash>#rep.json`,可复核。
- **调用缓存**:键 = `sha256(model|temperature|promptText)#repIndex`。同一 (模型,温度,prompt 文本,样本序号) 复用同一真实样本 → 既省钱又可复现;**改了 prompt 文本(v6→v7)哈希即变 → 自动重新真调**,不会拿旧样本冒充新结果。改断言不改 prompt → 复用同批真实输出重新判定(本轮已多次受益)。

## 2. 模型选型(任务 §占位B 要求写进报告)

| 角色 | 模型 | 说明 |
|---|---|---|
| 辩手/抽取/合议/治理(P1–P4,P6,P7) | `MiniMax-Text-01` | 账号可调的真实模型(新一代 MoE) |
| 裁判(P5)**异源** | `abab6.5s-chat` | 与辩手**不同模型族(上一代 abab 系列)**;同厂商跨代 |
| 多模型对比(by model) | `MiniMax-Text-01` + `abab6.5s-chat` | P1–P7 两个模型都跑,用于区分"prompt 缺陷"与"模型能力差" |

**为什么是 MiniMax 而非 Claude(关键诚实交代)**:
- 本环境的 `ANTHROPIC_API_KEY` **为空字符串**(Claude Code 走 OAuth 订阅令牌,子进程拿不到可用 API key,实测 length=0)。因此**无法对 Claude(这些 prompt 真正的目标模型)直跑**,也无法做真正的**跨厂商**异源裁判。
- 我没有就此停下:用户在仓库 `鹰眼原型/app/.env` 中留有一个**真实可用的 MiniMax key**(本人此前为"真·LLM"放入),实测可调多个模型族。MiniMax 是强中文模型,用于这套中文医保稽核 prompt 是合理的真实被测对象,且**跨厂商**(非 Anthropic 自家),对"自我偏好"这类测试反而更干净。
- **P5 异源**:任务明确认可"辩手 Sonnet 档 / 裁判 Opus 档"(同厂商不同模型)作为异源。本轮用 `MiniMax-Text-01`(辩手) vs `abab6.5s-chat`(裁判)落地同一精神——**同厂商跨代不同模型族**,诚实标注其**不是跨厂商**。

**这对结论的影响(必须随结论一起读)**:
- 在 MiniMax 上发现的失败,可能是 **(a) prompt 缺陷**(指令不够机械/可被绕过),也可能是 **(b) 模型能力差**(弱模型不follow指令、推理出错)。本轮用**双模型对比**与**人工核原始输出**来区分,并在报告里逐条标注。
- v7 的改法只针对 (a) 类(让指令更机械、更难被弱模型绕过);(b) 类如实记入 `OPEN_ISSUES.md`,并说明"需在目标模型 Claude 上复测才能定论"。
- **若要在 Claude 上回归**:补一个非空的 `ANTHROPIC_API_KEY`,在 `providers.js` 增加 anthropic provider 即可复用全部用例与断言(已预留)。

## 3. 评测口径(硬要求,任务 §3)

- **多次运行**:每用例每模型 **N=5**;报每条断言通过率(如 4/5)与全绿率,不做单次判定。
- **断言机器可判**:声明式断言引擎(`is_json`/`nonempty`/`equals`/`in`/`not_equals`/`custom`),复杂判定(金额去重、篡改未施加、位置一致)写成机读 `customChecks`。
- **JSON 提取稳健**:优先取 ```json 围栏内容 → 字符串感知的配平花括号扫描取最后一个对象 → 回退首{到末}。解析失败计入"JSON 不合规"而非崩溃。
  - ⚠️ 一个**重要的方法论教训**:首轮朴素提取器(首{到末})把模型思维链里回显的 `{...}` 片段一并吞入,**误报了一批 JSON 不合规**。修复提取器后(复用缓存、零额外花费)JSON 合规率显著回升。**说明:报"JSON 不合规"前必须先确认不是解析器的锅——否则就是在编造失败。**
- **温度**:准确性基线用低温(辩手 0.2,裁判 0.1 求一致);另跑高温(0.7)鲁棒性档,看 prompt 在抖动下是否仍守纪律。报告分档标注。
- **P5 真位置去偏**:同案 A/B **交换位置真跑两遍**,比对 verdict 一致性;独立脚本 `p5_swap_runner.js`(promptfoo 单条难表达)。归一化把"维持疑点→疑点""降为线索→线索""撤销/不成立→撤销"折叠。额外比对**模型自报的 `position_swap_consistent` vs 实测**(揭穿"自报戏")。
