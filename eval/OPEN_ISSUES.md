## B. 架构待办(prompt 层改不动,任务 §4 明确点名)

- **B1. P5 真位置去偏的最终保证**:prompt 层的 `position_swap_consistent` 是**自报**,真值只能由调用层把 A/B **交换跑两遍取一致**(不一致→降线索)来定。本轮 `p5_swap_runner.js` 已**把它测出来**(真双跑比对),但"修"它在调用编排层。
- **B2. 异源裁判编排**:裁判模型须与辩手模型异源。本轮用 MiniMax-Text-01(辩手) vs abab6.5s-chat(裁判)落地"同厂商跨代异源";**真正的跨厂商异源**(如 Claude 裁判 + 别家辩手)需调用层配置不同 provider。属编排层,非 prompt 文本。
- **B3. 传入规则定义本身畸形**(如要素 X AND 非X):应由 KB 准入做形式化校验,P2 prompt 触不到。

## C. 离线待办(统计/度量,单 prompt 解不了)

- **C1. 准标识 k-匿名度量**:P7 的 `reident_risk` 只能做"罕见组合"提示,真正的再识别风险需离线 k-匿名统计。
- **C2. 跨超长材料的实体合并**:同一药在第2块与第7块,需调用层分块后做实体一致性合并,P1 单 prompt 触不到(P1 只能标 `truncation_flag`)。
- **C3. cause_fingerprint 的聚类校准**:P7 输出的同因指纹归一,仍需离线聚类校准(近似但不同的根因可能误并/漏并)。
- **C4. 同因驳回比例阈值的基数**:P7 的"占比>15%且≥3次"门槛需接真实触发基数统计;本轮仅在用例的 recent_stats 里模拟基数。

## E. 本环境特有限制(影响"在目标模型上的结论效力")

- **E1. 无 Anthropic API key → 未在 Claude(目标模型)上回归**:`ANTHROPIC_API_KEY` 在本环境为空(OAuth 模式)。这些 prompt 是为 Claude 级模型设计的;本轮在 MiniMax 上的失败需在 Claude 上复测才能区分"prompt 真缺陷"与"MiniMax 能力差"。补一个非空 key 即可用同一套用例/断言在 Claude 上复跑(`providers.js` 已预留 provider 扩展点)。
- **E2. P5 跨厂商异源未做**:见 B2,本轮为同厂商跨代。

## D. 模型能力(非 prompt 可修;不应过拟合到 v7;需在 Claude 上复测才能定论)

> 判据:同一 v6 prompt,**主模型 MiniMax-Text-01 多数 5/5,而对照 abab6.5s-chat 系统性失败** → 失败来自模型能力,非 prompt 文案。这些**不改 prompt**。

- **D1. 弱模型不做"缺失/冲突"建模**:abab 在 P1-C2(missing_or_unclear 0/5)、P1-R3(conflicts 0/5)持续失败;主模型 5/5。
- **D2. 弱模型三态/除外判定弱**:abab 在 P2-C2(命中除外应不报,0/5)、P3-R1/R3/R4(无据/强反证仍坐实,0–1/5)失败;主模型 5/5。
- **D3. 时序比较是两模型共同弱点(部分能力天花板)**:P2-R3a(检测早于用药=合规应不报)主模型 3/5、abab 0/5。v7 已把比较要素求值写得更机械,但"先后判断"本身受模型推理能力限制——**v7 后若仍非满分,如实归此类**。
- **D4. 裁判一致性强依赖模型**:P5 中 MiniMax-Text-01 作裁判低温下仍高方差(逐rep≈0.72);abab 作裁判直接位置翻盘(C2/C4)。**裁判模型的选择本身是去偏的一部分**(呼应 B2 异源编排)。
- **D5. 独立除外扫描的有效性依赖模型**:P5-R2(控辩都漏的真实除外)主模型能独立扫到→撤销,abab 扫不到→维持疑点。"最后一道防线"在弱模型上失效。
- **D6. 思维链回显污染输出**:弱模型更易把推理过程/解释混进 JSON 前后(已由稳健提取器吸收;但极端情形仍可能影响)。
