# 鹰眼 EagleEye · 浏览器扩展(开单事前提醒)

**口号:安装即用,无需对接,无需调整任何调度。**

政策出处:"定点医药机构可以将两库置于本机构智能提醒等信息化系统中……将不合规行为消除在萌芽阶段"。
定位:医院**自装自用**提升服务效能的工具(不触碰"医保部门统一APP"边界,全程 B 端)。

> **成熟度(诚实标注)**:本扩展为**演示级最小可用原型**,验证"读开单页→本地引擎→事前提醒"的完整形态。
> 现挂载域为演示靶站 `mockhis.html`;适配真实 HIS 需按院方页面 DOM 配置表单选择器与 `matches` 域名,
> 并按院内网络收窄引擎 CORS。"安装即用"指演示靶站上的即装即演,非"任意 HIS 零配置生产可用"。

## 安装(演示)
1. 启动鹰眼本地服务:`cd prototype/app && node server.js`(:3700)
2. Chrome/Edge 打开 `chrome://extensions` → 开发者模式 → "加载已解压的扩展程序" → 选本目录
3. 访问演示靶站 `http://localhost:3700/mockhis.html` → 右上角出现"🦅 鹰眼已挂载"
4. 点"提交医嘱" → 插件先审方:16岁患者开左氧氟沙星 → 浮层弹出 AGE-101 提醒(两库年龄分层依据+政策原文)
5. 插件总入口: `http://localhost:3700/plugins.html` 可进入院端连播、监管小工具箱和院端运营看板

## 事前规则(开单时点纯输入可判)
- AGE-101 未成年用药年龄分层(14/18两档,走主引擎)
- F-001 性别—项目冲突(女开PSA/前列腺、男开宫颈/TCT → 明确违规,原生检测器)
- T-201 靶向药无基因检测(奥希替尼等未同开检测且诊断非突变阳性 → 可疑软提醒)
- B-201 超限定支付(白蛋白无指征/波生坦成人用 → 可疑软提醒)

诚实边界:主引擎里 F-001 无 checker、B-201/A-110 依赖检验值(事后),故这三条在事前层由 `engine/precheck-native.js` 以纯开单输入重新落地,除性别硬互斥外均诚实降级为"可疑"。事后明细审核仍以完整证据链为准。

## 结构
- `manifest.json` MV3;permissions=storage、options_page,host_permissions 默认本地引擎(可在设置页自配)
- `yingyan-precheck.js` 共享实现(与 MockHIS 内嵌演示通道同一份逻辑,源在 prototype/app/public/plugin/,浮层按三档分组)
- `content.js` 挂载徽标 + 按设置拦截"提交医嘱"先审方(有命中暂缓,不阻断诊疗——医生复核后可坚持提交)
- `options.html/js` 设置页:配引擎地址+测试连接、拦截开关、暂缓阈值(明确违规才暂缓 / 命中任意即暂缓)

## MockHIS 演示场景(靶站顶栏下拉切换)
S1 未成年+喹诺酮(AGE-101)· S2 女性开男性专属项目(F-001)· S3 靶向药无基因检测(T-201)· S4 超目录限定支付(B-201)· S0 干净件(合规放行)

## 适配真实 HIS
content script 的表单读取按"标签文本→相邻单元格"退化匹配(年龄/性别/临床诊断),
对未做 data-field 标注的老 HIS 表格同样工作;matches 域名按院方部署加白即可。

## 状态与部署边界
- 徽标会显示"本地引擎"或"内网引擎";当引擎地址不是 localhost/127.0.0.1 时,设置页会提示确认可信院内地址。
- 生产适配至少需要:院方 HIS 域名白名单、表单选择器/标签校准、院内 CORS 收窄、台账保留周期与管理员清空权限。
- 演示级不会替医生做最终决策:命中后暂缓提交,医生可采纳整改或记录理由坚持提交;后者流入监管重点审核台账。
