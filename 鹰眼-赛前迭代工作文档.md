## 一、定位与叙事(对外口径,一页讲完)

**一句话定位**:鹰眼是一套医保基金审核引擎,按国家智能审核监控规程的七个法定环节(数据采集传输 → 数据比对 → 违规筛查 → 明细审核 → 调查核实 → 违规处理 → 评估分析)将 Agent 逐一对位编排,同一引擎在两条产品线复用:

| 产品线 | 官方剧本 | 主打卖点 |
|---|---|---|
| **院端(事前/自查)** | 两库嵌入本机构信息化系统做事前提醒;对照 2026 年 12 大重点领域典型问题清单自查打报告 | 违规消除在"萌芽";主动自查可减少现场检查频次(政策激励) |
| **监管侧(事后全面)** | 全量结算单据智能审核全覆盖;疑点分诊解决"1 万条/月、医院过 1 遍医保过 3 遍";本地规则增补调参工作台 | 人少事多 → 一个人处理多案卷;人审判断反哺规则库("成熟一条应用一条") |

**技术身份**:确定性引擎(L1,毫秒级,对应"明确违规"自动拦截)+ LLM 多 Agent 合议(L2,对应"可疑"需听申诉)。分层本身与国家两库规则的"政策限定类/合理使用类"二分同构 —— 这是回应"通用模型谁都能做"质疑的核心论证:**不可替代的不是 Agent 拓扑,而是每个 Agent 的 harness 里装的法定规程、两库条目和判例。**

**叙事弹药(数字卡,易拉宝/口播用)**:
- 两库已发布至**第十六批**(2026-06-22),截至答疑时多数人还停留在"15 批"的认知;
- 2026 自查自纠 **12 大重点领域**(9+新增口腔/内分泌/精神医学),鹰眼深度专科命中其中 5 个(肿瘤/骨科/影像/麻醉/ICU);
- 三级医院每月疑似违规数据 **1 万+条**,医保部门要**过 3 遍**(初审/申诉复审/重审);
- 《监督管理条例实施细则》(2026-02 发布)第 35 条:DRG 入组错误损失=两病种支付标准**差额**,鹰眼直接算钱;
- 政策原文:"定点医药机构可以将两库**置于本机构智能提醒等信息化系统中**……将不合规行为消除在萌芽阶段" —— 这就是插件副产品线的合法性出处。

## 四、对抗合议三人格设计(Q8 定稿)

**同一底模,差异来自三个杠杆:上下文分区、工具分区、输出立场约束。**

| 人格 | 只能看到 | 弹药库 | 输出契约 |
|---|---|---|---|
| 医院辩护 | 病历/医嘱/检验(临床证据),**看不到规则引擎内部推理** | 申诉成功案例范式、《病历书写基本规范》、临床合理性论据 | 最强合规解释,每条辩点必须引用病历证据位置;被要求穷尽辩护 |
| 监管指控 | 规则命中详情+两库知识点+结算数据,**看不到完整病历叙事** | 两库条目、号令法条、指控书文书范式 | 指控书格式,每条指控必须挂法条/两库条目 ID |
| 专家裁定 | **只看两份陈述书**+KB2+判例库+裁量依据,不看原始数据 | KB2(NSCLC/骨科深喂)、曝光台判例、实施细则 | 强 schema:{成立/不成立/部分成立/证据不足转人工}+依据链;**依据必须是可解析 KB 条目 ID,引用不出即自动转人工**;钢人条款:裁定前必须先复述双方各自最强论点 |

协议:一轮立论 → 一轮质证 → 裁定,每轮 schema 约束+token 上限。全部辩论赛前真实跑出、结果预载(§二 Q11)。

## 六、演示与兜底(总原则②的落地)

**四级降级梯子(内嵌,无感切换,全程记日志)**:
LLM 调用 → ①超时/校验失败重试(≤2) → ②取预载真实结果 → ③纯确定性路径 → ④静态兜底视图。任何一级切换界面无感,内部日志进运维面板(G3)。

**PDF 乱包通道兜底**:乱包分三级脏度,现场默认演一二级;OCR 置信度低于阈值的字段标黄进人工确认态(这本身是产品功能,不是失败);扫码用户上传任意文件 → 类型白名单+大小限制+超时熔断 → 失败时返回"已收到,进入人工队列"的体面文案。

**实时性口径**:合议结果为赛前真实运行产出、现场预载;界面不标注缓存、不做假进度条;备快案卷支持现场任选实时重跑;被问即如实说明并当场重跑一条。

**二维码公网版红线**:仅自有公开/构造数据;与内网比赛数据物理隔离;限流。

**彩排两次**:断网模拟一次(验证四级梯子全程可走)、随机点击模拟一次(评委行为不可控)。
