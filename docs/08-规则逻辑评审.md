# 鹰眼 · 规则逻辑系统级评审 v1.0 —— 三个宏观漏洞 + 效率重构

> 触发:从整体审视规则逻辑——不是补单条规则,而是看规则之间、规则与引擎之间的结构性问题,扣细节、扣效率。
> 编制日期:2026-06-12。已同步主文档 v1.5 与 Cowork Prompt。
> 一句话结论:**单条规则已经写得不错,但"规则的集合"还不是一个系统**——缺合议、缺覆盖度自检、缺冲突消解,执行上还有大量重复劳动可省。

---

## 第一部分:三个宏观漏洞(决定可信度的生死)

### 宏观漏洞 ① ⭐⭐ 疑点去重与合并:同一笔钱被多条规则重复报 —— 当前最危险的问题

**现象**:一笔人血白蛋白违规结算,B-201(超目录限定支付)、A-110(范围外费用纳入结算)、A-108(无医嘱/执行记录)可能同时命中——三条规则逻辑上都成立,但指向**同一笔 1280 元**。当前"规则独立判定→汇总"的架构会把它输出成 3 条疑点、涉及金额累加成 3840 元。

**为什么是生死问题**:对质现场,稽核员拿 3 条疑点找医院,医院一句"你把一件事算三遍、金额翻三倍想讹人"——整份报告的可信度当场归零。**误报里最致命的不是"报错",而是"重复报 + 金额虚高"**,因为它让对方抓到"你在夸大"的把柄。

**根因**:规则体系缺一个**裁决合议层(Finding Reconciliation)**。规则可以多对一(多条规则命中同一事实),但输出必须一笔违规一条主疑点。

**方案——疑点聚类与主从合并**:
1. **聚类键**:以"涉及的核心费用行 ID 集合"为键。指向同一笔/同一组费用行的多个 raw findings 聚为一簇。
2. **簇内定主**:簇内按"违规定性的确定性 + 证据完整度 + 罚则严重度"选一条做**主疑点**,其余降为**佐证视角**附在主疑点下(不独立计金额)。例:白蛋白簇 → 主疑点 B-201(限定支付,定性最准),A-110/A-108 作为"另从范围/医嘱角度亦可佐证"列示。
3. **金额一次性归集**:`amount_involved` 按费用行**去重求和**,一笔钱只算一次。报告头部"涉及金额合计"必须是去重后的真实暴露金额。
4. **多重定性保留但不重复计数**:同一笔钱可能既是"串换"又是"超限定"——定性都列出供稽核员选择对质角度,但金额、疑点计数只算一次。

**对质价值反转**:合并后,主疑点 + 多视角佐证反而是**更强的对质材料**——"这笔钱从限定支付、目录范围、医嘱证据三个角度看都站不住",比三条孤立疑点有力得多。**把潜在的"夸大"漏洞,变成"铁证如山"的优势。**

**新增架构层**:L5 验证层之后、L6 输出之前,插入 **Reconciliation 合议层**:去重 → 聚类 → 定主从 → 金额归集 → 多定性合并。这是 07 号文档七层架构的必要补位。

### 宏观漏洞 ② ⭐ 覆盖度盲区:无法回答"这份材料我查全了吗"

**现象**:当前是"规则主动扫描命中即报"。但稽核的本质问题是**否定式的**——"这份材料**还有什么没查**?"现在系统答不出来。一份材料跑完 41 条规则,没报疑点,到底是"真干净"还是"没有规则覆盖到那类违规"?无从区分。

**为什么重要**:① 稽核员需要知道覆盖边界才敢签字("已用 X 类规则核验,未覆盖 Y 类需人工");② 监管验收会问"漏检率",没有覆盖度模型就没法回答;③ 这正是产品"高精确率（低误报）为先 + 漏判兜底（优先通路全量扫描+人工复核+国家大数据）"承诺的另一面——漏报也要**已知地漏**,而非**未知地漏**。

**方案——覆盖度清单(Coverage Manifest)**:
1. 把"应查维度"显式化为清单:费用-医嘱一致性、费用-诊断相符性、限定支付、收费规范(重复/分解/超标)、住院行为、支付方式、资质数据……每个维度标注"本次是否有规则实际执行、执行了哪几条、是否因材料缺失而跳过"。
2. 输出报告附**覆盖度声明**:"本次核验覆盖 8 个维度共 X 条规则;因缺护理记录,A-105/A-108 部分降级为线索;耗材类规则(本院无植入耗材)未触发。"
3. **材料完整性前置检查**:开跑前先核材料包齐不齐(有无费用清单、医嘱单、护理记录、检验、病理),缺失项直接影响哪些规则可执行——在报告里显式列出,而非默默跳过。

**pitch 价值**:"我们不仅告诉你查出了什么,还告诉你查了哪些、哪些没查到、为什么"——这是稽核员敢用、监管敢认的前提。可信度从"报得准"升级到"边界清"。

### 宏观漏洞 ③ 规则冲突与优先级:命中互斥时谁说了算

**现象**:存在规则会得出**逻辑冲突**的结论却无消解机制:
- **互斥定性**:某项目同时疑似"串换"(A-107,实际做了 A 收了 B)和"虚记"(A-108,根本没做)——两者互斥(要么做了别的、要么没做),不能同时成立。
- **包含关系**:C-301(分解住院)成立时,该次住院内的某些 C-302(挂床)疑点可能是分解住院的子现象,不应平行计。
- **特例优先**:DRG 特例单议病例、放化疗周期病例,应豁免某些规则,但当前豁免散落在各规则的 exclusions 里,没有统一仲裁。

**方案——规则关系图谱 + 优先级仲裁**:
1. **声明规则间关系**:在规则 Schema 增 `relations` 字段——`mutually_exclusive_with`(互斥)、`subsumes/subsumed_by`(包含)、`overrides/overridden_by`(优先)。例:A-107 与 A-108 互斥;C-301 subsumes 同住院段的 C-302。
2. **仲裁规则**:互斥时取证据更强者,弱者转为"备择定性"列出;包含时只报父、子作为父的明细;特例豁免统一在仲裁层处理,不依赖各规则自己记得写 exclusions。
3. **全局豁免清单(Suppression List)**:DRG 特例单议、临床路径变异、急诊抢救、放化疗周期等"合法的异常",集中维护为豁免规则,在仲裁层统一应用——避免同一豁免在 N 条规则里重复写、漏写。

## 第二部分:规则覆盖结构盘点(宏观看缺什么类)

> 站在"违规全景"高度,看现有 6 大类是否有系统性缺类。

**现有**:F 基础逻辑 / A 收费合规 / B 诊疗合理 / C 住院行为 / D 支付方式 / E 资质数据。

**结构性缺口(建议补类,非补单条)**:

| 缺口 | 说明 | 建议 |
|---|---|---|
| **时序/跨就诊维度弱** | 现规则多为"单次就诊内"判定;分解住院(C-301)、重复开药(B-205)虽涉及跨次,但缺独立的"时间序列"规则族:如同一患者短期内同项目跨机构重复、慢病规律配药节奏异常突变、住院-门诊费用腾挪 | 设 **G 类:时序与跨就诊**(黑客松可仅占位,数据底座依赖批量模式) |
| **金额/统计离群无规则** | 现规则是"逻辑命中式",缺"统计离群式"——某项目单价/用量显著偏离同类病例分布(无需先验规则即可发现新型违规) | 设 **H 类:统计离群**(需群体数据,P2;但架构要预留,这是发现"未知违规"的唯一途径) |
| **门诊场景几乎空白** | 现规则重住院。但门诊统筹、门诊大病、药店购药(回流药、串换、超量)是 2026 监管重点之一 | A/B 类规则补 `applies_to:门诊` 变体;药店场景单列 |
| **耗材维度偏薄** | A 类侧重收费,B 类侧重药品,医用耗材的"无指征植入、数量与手术不符、高值耗材串换低值"覆盖不足 | A/B 类补耗材专项条目(骨科备演场景正好用) |
| **服务质量/服务不足** | 现规则抓"多收多治",对"少做(收了费没做够)"覆盖弱(D-403 仅占位) | 与临床路径 KB2 结合,补"路径关键环节缺失"规则 |

**判断**:黑客松不必全补,但 pitch 架构图里这 6+2=8 类要画全(G/H 标"路线图"),体现"违规全景"的完整认知——评委会问"你这规则覆盖得全吗",有全景图就答得上。

## 第三部分:执行效率重构(扣效率)

> 把"规则集合"当成一个要跑上万份材料的引擎,看哪里有重复劳动、可省的计算、可并行的结构。

### 效率 ① 共享事实计算,杜绝重复取证

**问题**:多条规则各自向病历"问同一个问题"。A-105/A-108/T-204 都要"护理等级是多少""某药用了几次";B-201/B-202/T-201 都要"诊断是什么""有无某检验"。当前每条规则独立调 LLM 重复抽取,是最大的隐性浪费。

**方案**:7 号文档的**事实层**正是解药,但要明确规则与事实的契约——规则**不允许**自己去原文抽取,只能**声明所需事实键**(declares `requires: [护理等级, 头孢呋辛执行次数, 白蛋白检验值]`),由事实层统一抽取一次、全规则共享。规则逻辑退化为"在已抽取事实上做判断",纯函数、可单测、零额外 LLM 调用。**这条是效率的总开关**:抽取从"规则数 × 费用行数"次降到"每份材料一次"。

### 效率 ② 三级短路执行(L1→触发器→L2)

**问题**:贵的 L2 语义判定不该无条件跑。

**方案(细化 07 号触发器路由)**:
- **第 0 级 材料门**:材料缺失→相关规则直接标"无法执行"(进覆盖度清单),不浪费算力。
- **第 1 级 L1 确定性筛**:F 类 + 结构化谓词在事实层上跑(纯代码,微秒级)。性别/时间/数量/互斥先排雷。
- **第 2 级 触发器谓词**:每条 L2 规则的廉价前置条件(T-201 触发器=`存在靶向药费用行`)。不命中→根本不进 LLM。
- **第 3 级 L2 语义判定**:仅对触发器命中的"候选"调 LLM,且**同规则多候选合并一次调用**。
- 量化预期:典型住院材料,41 条规则中实际进 LLM 的通常 ≤10 条,LLM 调用次数较朴素实现降一个数量级。

### 效率 ③ 判定结果缓存与增量

- **规则-事实指纹缓存**:`hash(规则版本 + 所依赖事实值)` 作键缓存判定结果。补传一份材料、或规则微调,只重算指纹变化的部分,不全量重跑(对批量飞检意义重大)。
- **知识库检索缓存**:同一目录条款、同一药品限定备注会被多份材料反复查,KB 检索结果按 `(条款ID, as_of日期)` 缓存。

### 效率 ④ 规则编译与冷热分层

- **规则编译**:把规则的结构化部分(触发器谓词、事实依赖、关系声明)在加载时**编译成执行计划**,而非每次解释 YAML。
- **冷热分层**:高频高命中规则(A-108 虚记、A-101 重复收费)优先执行并重点优化;低频规则(E-501 资质)按需加载。
- **规则分组并行**:无依赖关系的规则簇可并行判定;有 `subsumes` 关系的串行(先父后子)。

## 第四部分:单条规则的细节打磨(扣细节)

> 抽查现有规则,找定义层面的精度漏洞。这些不修,现场会被"钻空子"。

| 规则 | 细节漏洞 | 修法 |
|---|---|---|
| **A-109 / T-204 超量** | "结算数量 > 医嘱量×天数"未扣**合理损耗与最小包装规格**。如药品按支供应,医嘱 1.5g 但规格 0.75g/支,需 2 支无浪费;胰岛素、生物制剂有开瓶损耗 | 引入 `规格向上取整 + 损耗系数(按剂型参数化)`;差额需超过"一个最小包装"才报,否则噪声 |
| **C-301 分解住院** | 间隔"≤N 日"是硬阈值,N=15 与 N=16 天差一天就翻案,经不起申诉;且未区分"计划性分次"(化疗周期)与"同病连续" | 阈值改为**带缓冲区的分级**:核心区(≤7 天高度可疑)/ 观察区(8–15 天需更多佐证)/ 豁免白名单(周期治疗);输出注明落在哪个区 |
| **A-105 护理等级** | "护理记录频次不满足等级"需明确**判定标准来源**,否则是 LLM 主观感受 | 频次标准必须引 KB2/地方护理规范原文条款,无明确标准的降为线索 |
| **B-201 限定支付** | "限重症/抢救"等定性条件**无量化口径**时,LLM 判定不稳;且限定条款解析易遗漏"或"关系(满足任一即可) | 限定条款解析为**显式布尔结构**(AND/OR 树),无量化口径的条件标记"需人工研判"→线索;不强行二分 |
| **A-108 虚记** | "材料中无对应执行记录"的**阴性证明范围**未界定。护理类自动执行项、套餐展开项易误判 | 明确"已核对单据范围"白名单 + 套餐先展开后比对(已在 03 号提及,提升为硬约束);阴性结论必须可复核 |
| **F-004 单日超量** | 吸氧>24h/日等,未考虑**跨日连续治疗**的归集口径(23:00–次日 02:00 算哪天) | 计量按"治疗事件"而非"自然日"归集,定义事件切分规则 |
| **T-201 靶向药** | "明确作用靶点药物"清单需**版本绑定说明书**;同一药不同适应症检测要求可能不同 | 靶点检测要求按 `(药品, 适应症)` 二元组查,非按药品单维;绑定 2025 版指导原则版本号 |
| **D-401 高套** | "编码无病历支持"需对接分组器看**是否真的影响入组/权重**——不影响分组的编码瑕疵不构成套高 | 仅当编码差异**导致 DRG 组/权重变化**时才报(用 OpenDRG 跑两次对比),纯编码错误转质控提示而非违规 |

**通用细节原则(写入规则编写规范)**:
1. **所有阈值带容差**:硬阈值一律改为"核心区/观察区"分级,杜绝"差一天差一块就翻案"。
2. **所有定性条件标注口径来源**:可量化的给阈值,不可量化的(如"重症")标"需人工研判"→线索,不让 LLM 硬判。
3. **所有"无记录"类判定界定阴性范围**:必须说明"核对了哪些单据、哪些属合理无需记录",阴性结论可复核。
4. **所有数量类判定扣规格与损耗**:差额需超过一个最小包装单位才报。
5. **金额计算口径统一**:违规金额 = 违规部分而非整笔(超量只算超出部分的钱,不是整条费用)。这条直接关系宏观漏洞①的金额准确性。

## 第六部分:三句话总结

1. **最危险的洞是缺合议层**:41 条规则会对同一笔钱重复报、金额虚高,对质现场这是致命把柄。补一个 Reconciliation 层(聚类→定主从→金额去重→多定性合并),把"夸大漏洞"反转成"多视角铁证"。
2. **缺覆盖度与冲突仲裁两个系统级机制**:要能回答"查全了吗"(覆盖度清单)、要能消解"互斥/包含/特例"(规则关系图谱+全局豁免清单)——规则才从"一堆"变成"一个系统"。
3. **效率总开关是事实层契约**:规则只声明所需事实键、由事实层统一抽取一次,配合三级短路(材料门→L1→触发器→L2)与缓存,LLM 调用降一个数量级;细节上所有阈值带容差、数量扣规格、金额只算违规部分——让规则既快又经得起申诉。
