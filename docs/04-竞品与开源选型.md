## 1. 竞品拆解：恒识 Evercog（同赛道）

> 来源：模法黑客松 S4 医保智能体专场组队招募海报。产品名"恒识 医保基金监管智能作业平台"，同属基金监管赛道。

### 1.1 值得借鉴的三个架构思路

**① 规则沉淀闭环（最值得抄）**
- 他们的核心工作流是闭环：监管目标输入 → Agent 拆解任务 → 调度数据/规则/分析模型 → 自动召回全量明细 → 政策匹配风险分级 → 生成可溯源证据链 → **人工复核标注 → 经验反向沉淀规则**。
- 关键差异：最后两步形成回路。鹰眼当前是"跑规则→出报告"的单向流，他们多了"稽核员复核结果回流成新规则/调参数"。
- **鹰眼行动**：把这个回路提到核心架构。稽核员在复核疑点时点"成立/不成立/误报"，这个标注应当：a) 进入 few-shot 案例库提升后续判定；b) 对反复误报的规则自动建议调整除外条件或阈值。这把主文档里排在 P2 的案例库提到了工作流主干。

**② 多维度风险画像层**
- 他们第2层是"机构/医生/科室/药品/病种多维度风险画像与监管指标体系"，技术合伙人是大数据架构师（实时离线指标体系）。
- 关键差异：鹰眼当前是单份材料包稽核，缺跨材料的画像维度。而鹰眼的 L3 线索规则（如某医生某项目长期异常、某科室某药使用率离群）正需要这个数据底座。
- **鹰眼行动**：承认这是他们的强项牌（大数据底座）。鹰眼黑客松阶段不与之硬碰，但在架构图中预留"画像层"接口，pitch 时说明"单材料语义稽核 → 可向群体画像扩展"，避免被评委问到时显得没想过。

**③ 落地路线选骨科耗材打头阵**
- 他们 V0.1 演示版选**骨科高值耗材**，V1.0 才覆盖"高值耗材+重点药品"。
- 推测理由：高值耗材违规是"数量对不对、内涵含不含"，证据硬、判定确定、现场演示不易被专家挑战；而靶向药"指征判断"依赖医学专业性，被专家质疑的风险高。
- **鹰眼行动**：这是一个需要重新权衡的决策点 —— 见下文 1.3。

### 1.2 他们的弱点 = 鹰眼的差异化优势（守住别丢）

| 维度 | 恒识（海报所示） | 鹰眼 | 鹰眼优势点 |
|---|---|---|---|
| 可信度承诺 | "高价值线索低误判"（结果导向） | "引不出原文不输出 + 宁漏报不误报"（机制导向，三要素门禁） | **机制化可信度**：在对质场景，误报让稽核员白跑、得罪医院，机制承诺比能力承诺更打动人 |
| 落地深度 | 全是层级框图，无具体规则/证据样例 | 41条带证据集的规则 + 病历包预埋点设计 | **规则落地深度**：框架谁都会画，规则写不写得出是另一回事 |
| 语义稽核 | 强调"规则引擎层（阈值/频次/组合/趋势）" | 明确"超越字段比对的语义级稽核" | 鹰眼直击非结构化病历，他们规则引擎措辞偏传统字段比对 |

> pitch 策略：**抄他们的"规则沉淀闭环"和"画像层"补齐架构完整性，但全程强调鹰眼的"机制化可信度"和"已落地的规则深度"**。框架对等之后，比的是谁真能跑出可对质的疑点。

### 1.3 ⚠ 需要你拍板：Demo 选骨科还是肿瘤？

| | 肿瘤（当前选择） | 骨科耗材（竞品选择） |
|---|---|---|
| 故事性 | 强（靶向药、基因检测，叙事完整） | 中（耗材数量、内涵） |
| 证据硬度 | 中（指征判断依赖医学专业，易被专家挑战） | **高**（数量/内涵比对，判定确定，难反驳） |
| 体现语义稽核 | **强**（读病历找基因检测、病理、线数） | 中（部分仍是结构化比对） |
| 现场风险 | 较高（专家可能质疑用药指征判断） | **低**（耗材违规对错分明） |
| 与竞品区隔 | **高**（避开他们的主场） | 低（正面撞他们 V0.1） |

**我的建议**：维持肿瘤为主线，但**把骨科耗材做成"备演场景"**。理由：① 肿瘤最能体现"语义级稽核"这一核心创新，正面区别于竞品的规则引擎路线；② 但保留一个骨科耗材的硬证据案例作为 backup，万一现场评委里有临床专家对肿瘤指征判断提出异议，立刻切到骨科耗材展示"对错分明"的稽核——证据硬度托底。两手准备，攻守兼备。最终由你定，若团队医学资源薄弱、怕现场被挑战，可直接换骨科为主线。

## 2. 多 Agent 讨论机制：该不该引入、怎么落

### 2.1 结论：该引入，但要克制

学术证据明确支持，且**恰好命中医保稽核的核心诉求（可信度）**：
- "society of minds"式多 agent 辩论被证明能提升事实准确性、减少幻觉和谬误答案（Du et al. 2023，arxiv 2305.14325）。
- 在**医学、法律等关键领域**，已有研究用对抗辩论+投票实现多 agent 交叉验证，并据此判断何时需要外部检索（MDPI Applied Sciences 2025）。
- 最贴近鹰眼的是 **Tool-MAD（事实核查辩论框架）**：给每个 agent 配检索工具，通过反复更新证据、互相挑战结论来精炼推理、抑制幻觉，引入自适应查询让 agent 按辩论进展迭代检索，在医学 QA 下稳健（arxiv 2601.04742）。
- 经典结构：**两辩手 + 一裁判**，辩论主要在辩手间，几轮无果裁判介入裁决；特别适合**是非（Yes/No）型问题**——稽核疑点恰好是"成立/不成立"二元判定（arxiv 2407.20505）。

**⚠ 关键警告**：过多辩论轮次不仅增加成本，还会因给模型更多生成机会而**放大幻觉**（MDPI 2025）。所以辩论必须硬性封顶轮数，不是越多越好。

### 2.2 鹰眼的多 Agent 设计（"模拟对质"机制）

把医保稽核现场的真实博弈搬进 agent——稽核员出疑点、医院申诉、专家裁定。这天然契合三辩手结构，且直接服务"结论可用于与医疗机构对质"的产品承诺：

```
                    ┌─────────────────────────────────────┐
                    │  ① 稽核 Agent（控方/Prosecutor）      │
                    │  立场：找违规。跑规则库，提出疑点，    │
                    │  从材料包取证，给出初步定性与条款引用  │
                    └──────────────┬──────────────────────┘
                                   │ 疑点 + 证据 + 条款
                                   ▼
                    ┌─────────────────────────────────────┐
                    │  ② 申诉 Agent（辩方/Defender）        │
                    │  立场：为医院辩护。检查除外情形、     │
                    │  寻找反向证据、质疑证据链完整性、      │
                    │  指出是否属合理诊疗/医学必要          │
                    └──────────────┬──────────────────────┘
                                   │ 反驳 / 认可
                                   ▼
                    ┌─────────────────────────────────────┐
                    │  ③ 裁判 Agent（Judge）                │
                    │  立场：中立定性。综合双方论点，       │
                    │  核验三要素是否齐备，裁定：           │
                    │  疑点(闭环) / 线索(存疑) / 撤销(误报) │
                    └─────────────────────────────────────┘
```

**机制要点（直接映射产品原则）**：
1. **辩论只对"高风险疑点"启动**，低风险或 L1 确定性规则命中的不辩（省成本、防过度）。对应主文档分层：L1 不辩，L2 高风险才辩。
2. **辩论轮数硬上限 = 2 轮**（辩手各陈述一次+一次反驳），无果即裁判定夺。直接落实学术警告。
3. **裁判的裁决标准就是三要素门禁**：申诉 agent 若能指出任一要素缺失或有效反向证据，裁判降级为线索或撤销 → **这正是"宁漏报不误报"的工程实现**。申诉 agent 本质是误报过滤器。
4. **申诉 agent 的反驳理由进入复核界面**：稽核员看到的不只是疑点，还有"医院可能这样辩、我方如何应对"——直接服务"可用于对质"的承诺，比单 agent 报告更实战。
5. **可解释性增强**：辩论过程本身就是推理过程的展开，比单 agent 的链式推理更经得起审视。

**演示价值**：这是 pitch 里区别于所有竞品的杀手锏。竞品讲"风险分级"，鹰眼能现场展示"控辩裁三方对质"——把稽核现场的博弈预演了一遍，稽核员拿到的是已经过对抗检验的疑点。对应主文档病历包里的 #6（升白针，合理性争议）正好用来演示辩论如何把它从"疑点"降为"线索"。

### 2.3 落地成本提示
- 黑客松阶段可**简化为单模型多角色 prompt**（同一模型扮演三个角色顺序发言），无需真起三个 agent 进程，先验证机制有效性。
- 成本敏感：辩论使 token 消耗约 3 倍（三角色），故只对高风险疑点启动是必须的。
- 进阶：成熟后用 AutoGen 的 GroupChat 或 LangGraph 的状态图实现真多 agent（见第3节）。

## 3. 开源选型：哪些能直接用

> 调研结论：鹰眼**不必从零造轮子**。三个核心工程难题（文档解析、证据链、DRG分组）都有成熟开源，KB1 甚至有官方现成数据。

### 3.1 ⭐ 最高价值发现：国家医保局官方规则库已开源公开

- **《医疗保障基金智能监管规则库、知识库（2025年版）》**：国家医保局 2025 年以来已通过官网公开发布 **8 批监管规则、近 2 万条知识点**；出版物全量收录**国家版智能监管规则 88 类、知识点 24.7 万条**，所有知识点已对应医保编码，可直接部署应用，且药品知识点已按 2025 版目录更新。
- **对鹰眼的颠覆性意义**：KB1 不用从零写。**官方版规则库做底座（管刚性约束/字段比对），鹰眼 41 条自研规则补语义级稽核**。定位瞬间清晰：
  - 官方规则 = 字段比对、刚性约束（事前提醒拦截那类）
  - 鹰眼规则 = 读病历的语义判定（超越字段比对，正是创新点）
- **行动**：到 nhsa.gov.cn 抓取已公开的 8 批规则与知识点（art_14_19660.html 及历批公告），结构化入 KB1 作为基线；鹰眼自研规则标注为"语义增强层"。pitch 时这是巨大加分——站在官方规则肩上做增量，而非另起炉灶。
- 来源：nhsa.gov.cn 2026-02-12《规则库知识库2025年版出版发行》。

### 3.2 ⭐ 文档解析 + 证据链：RAGFlow（直接用作底座）

- **infiniflow/ragflow**，Apache 2.0（可商用），GitHub 7万+ stars。**几乎为鹰眼量身定做**：
  - **DeepDoc 解析层**：布局感知，原生处理表格、扫描件 PDF、多栏文档（不是按字符切）；自带混合检索（向量+BM25）。→ 直接是鹰眼技术路线第①层"原生理解扫描件与表格"。
  - **证据链可追溯（鹰眼命门）**：点击引用跳转到源文档精确位置——具体段落/表格单元格/某页。→ 直接实现"证据定位到费用行/病历原文"。
  - **私有化自托管**：源文档不出内网。→ 对齐"数据不出域"。
  - 现成 **laws/manual 法规类切分模板**（切《条例》《目录》）、医疗顾问 agent 模板、GraphRAG 支持。
  - 2026-03 起提供 OpenClaw/Skill 接入。
- **行动**：用 RAGFlow 作为 RAG 底座承载 KB1/KB2，省掉自建文档解析+检索+引用定位的大量工程。鹰眼聚焦于上层的规则推理与多 agent 机制。
- 来源：github.com/infiniflow/ragflow。

### 3.3 ⭐ DRG 分组基准：OpenDRG（D-401 依赖）

- **github.com/OpenDRG**，永久免费。建立了 CHS-DRG 1.0/1.1、CN-DRG 2018、ZJ-DRG 等分组方案知识库，收集全国三十多个城市分组规则/权重/支付标准，提供 Java/C#/Python/JS 四语言分组器，可嵌入 HIS/EMR/病案系统，**含江苏徐州等地区版本**。
- **行动**：鹰眼 D-401 高套分组规则需要分组器做基准（病历实际应入哪组 vs 上报组）。OpenDRG 直接提供，且有江苏版本契合锚定。
- ⚠ 注意：需核验是否已更新到 DRG/DIP **2.0 版**分组方案（主文档2.6节，2025年起统一执行）；若仍为1.x需自行升级或叠加2.0规则。

### 3.4 多 Agent 框架：三选一

| 框架 | 定位 | 适合鹰眼场景 | 选型建议 |
|---|---|---|---|
| **AutoGen**（microsoft） | 对话式多 agent，GroupChat 由 Manager 调度发言顺序 | **多方辩论/共识**的对话模式最丰富 | 多 agent 辩论首选；但微软已转维护模式（转向 Microsoft Agent Framework）|
| **LangGraph** | 状态图，可循环/分支/断点续跑 | 生产级有状态系统最经得起考验；控制流精确 | 要精确控制辩论轮数/状态时选它 |
| **CrewAI** | 角色化（agent 有 role/goal/tool），上手最快 | 快速原型团队协作 | 黑客松快速验证可用 |

- **黑客松建议**：先用**单模型多角色 prompt** 验证辩论机制（2.3节），不引框架；若要真多 agent，**LangGraph** 最适合（能精确控制"2轮封顶+裁判介入"的状态流转），CrewAI 上手更快可作备选。
- 现成参考实现：github.com/NishantkSingh0/Multi-Agent-Debates-LangGraph（Scientist↔Philosopher↔Judge 的辩论循环，StateGraph 实现，可改造为控辩裁结构）。

### 3.5 医疗知识图谱（可选增强，非必需）

- **liuhuanyong/QASystemOnMedicalKG**：以疾病为中心的医药领域知识图谱+问答。可作 KB2 临床知识的补充结构化来源（疾病-症状-药品关系），但与官方指南/说明书相比权威性弱，仅作辅助，黑客松非必需。

### 3.6 选型总览：鹰眼应该"组装"而非"重造"

```
鹰眼 = RAGFlow（文档解析+证据链+RAG底座）
     + 国家医保局官方规则库2025版（KB1基线，88类规则/24.7万知识点）
     + OpenDRG（DRG分组基准，江苏版）
     + 鹰眼自研41条语义规则（核心创新：超越字段比对的语义稽核）
     + 多agent控辩裁机制（LangGraph，可信度杀手锏）
     + 鹰眼自建肿瘤病种包KB2（用2025版抗肿瘤指导原则）
```

**这个组装策略的 pitch 价值**：① 用官方规则库做底座 = 权威性背书 + 不重复造轮子；② RAGFlow 解决文档解析这个最脏最累的活，团队聚焦稽核智能；③ 鹰眼真正的护城河收窄到两点——"语义级规则"和"多agent对质机制"，资源集中，黑客松周期内能做深。

## 5. 给你的三句话总结

1. **竞品恒识**：抄它的"规则沉淀闭环"和"画像层"补架构，守住你"机制化可信度"和"规则落地深度"两个它没有的优势；demo建议肿瘤主线+骨科备演。
2. **多agent**：值得做，设计成"控辩裁三方对质"（稽核/申诉/裁判），申诉agent即误报过滤器=宁漏报不误报的工程实现，辩论2轮封顶防幻觉，这是pitch杀手锏。
3. **开源**：别重造轮子。国家医保局官方规则库已开源（88类/24.7万知识点）做KB1基线，RAGFlow做文档解析+证据链底座，OpenDRG做DRG基准，鹰眼只需深耕"语义规则+多agent机制"两个真护城河。
