### 修法(P4_defense_v72.txt,不覆盖 v7 基线)
在 `evidence_gap_exists` 判据加一条**机械硬规则**专治「缺失即违规」型倒置:
> 当规则本身是「用了X却【无】Y(检测/病理/指征/执行记录)」——违规由 Y 之缺失构成时:
> 事实相应数组为空、且控方 counter_evidence 已声明「已检视…未见Y」→ evidence_gap_exists 必须=false。
> 「控方未见Y/未排除外院Y/可能有外院Y」一律=举证责任倒置+臆造,禁止写入 rebuttals。
> 控方需证「用了X且院内无Y」,无需证「全世界都无Y」;举证责任在医院出示Y。

同时在除外路径加「防『缺失即除外』臆造」:要求文档原件的除外,该文档必须真实在事实中方可命中。

## 二、金标准去自评化

### 循环性确认
`case_*/expected_findings.json` 原写明 `"gold_standard_note":"Oracle 引擎自动生成"` ——
**recall 检查是拿引擎输出比引擎自己的快照(循环自评)**。
但 G0 干净档红线只依赖 `case_meta.embedded_violation_count`(埋点时人工声明,`===0`),**不循环**。

### 去循环做法
1. **独立地面真值清单** `prototype/data/ground_truth_manifest.json`:按【埋点设计意图】(案名 + evc + 病历研读)
   逐案声明 `is_clean / planted_suspect(必抓疑点)/ planted_clue(必抓线索)`,**与引擎输出解耦**。
2. **独立召回 harness** `yhf/harness/manifest-recall.js`,并接入 `yhf/index.js` + `yhf/gate.js`:
   - **G0b_clean_zero_fp(阻塞)**:15 个声明干净案必须 0 疑点 —— 与引擎埋点计数**互不依赖的第二道零误报红线**。
   - **recall_floor(报告态,非阻塞)**:35 条设计 floor 规则必须被召回;同既有 recall 口径不阻塞。
3. **快照诚实化**:31 份 expected_findings.json 的 `gold_standard` 改 false、标 `snapshot_role=engine_regression_snapshot`,
   注明「非独立金标准,独立真值见 ground_truth_manifest.json,本快照仅供 shadow/回归漂移探测」。

### 门禁结果
- **G0b clean zero FP(独立复核):✅ PASS**(15 干净案全 0 疑点)
- **recall floor:✅ 全召回**(35 floor 规则)
- overall:✅ PASS

### 独立真值当场抓到一处真漂移(去循环的价值实证)
`violation_light_f006`:规则库更新把 **F-006 从「银杏二萜内酯超疗程」重定义为「限频项目超频次」**,
案卷埋的是超疗程违规 → 与新 F-006 语义脱钩 → 不再命中。
- 引擎旧快照会被重生成而**掩盖**此脱钩;独立 manifest **永久诚实标记**为 `pending_realignment`(规则库在飞态,不阻塞门禁)。
- **待规则线处理**:要么把 F-006 案卷重对齐到新语义,要么给「超疗程」另立规则号后回填 floor。

### 口径更新
- 旧:「金标准是 Oracle 引擎自评(引擎自评风险)」
- 新:「**双独立真值源**:①埋点人工声明 embedded_violation_count(G0)②设计意图 manifest(G0b + recall floor);引擎快照已降级为回归漂移探测,不再充当金标准」
