## 0. 先答"还有什么好思路"：五个补充

1. **一套引擎两种模式（院端自查 = 现成的第二市场）**：2026 年政策强制全国所有定点医疗机构 4 月底前完成自查自纠，且逐年常态化。同一套鹰眼引擎切两个面板——"稽核模式"（监管侧找违规）和"体检模式"（院端医保办自查防飞检）。院端付费意愿强（怕被罚 1–5 倍）、客户数量大（万级机构），恒识也在打"医院医保办"。黑客松 pitch 里讲清这一点，商业故事立刻完整。
2. **金额 × 置信度排序**：疑点列表按"涉及金额 × 证据置信度"排序输出，稽核员先处理 ROI 最高的。实现成本极低，体验提升极大——这是 Alaffia 等国际厂商验证过的产品直觉（优先审高金额住院账单）。
3. **规则即代码（Rule-as-Code）**：每条规则像软件模块一样自带单元测试（≥3 阳性用例 + ≥3 阴性干扰用例）、版本号、评审记录、影子运行指标——这是第 4 节准入机制的底座，也是"规则本身也有证据链"这句 pitch 金句的来源。
4. **对质预案输出**：控辩裁机制（04号文档）的副产品——申诉 agent 的反驳话术直接随疑点输出给稽核员，即"医院大概率会这样辩，建议这样应对"。把 AI 初筛升级为 AI 对质参谋。
5. **药品追溯码证据轴（前瞻预留）**：2025 年 7 月起药品销售强制扫追溯码结算，追溯码数据未来可作第四验证轴（进销存一致性、倒卖回流药识别）。黑客松不做，但架构图预留接口、pitch 提一句，显前瞻。

## 1. 竞品扫描（第二轮）

### 1.1 ⭐ 国际对标：Alaffia Health —— "美国版鹰眼"，pitch 最佳对标

最像鹰眼的公司，没有之一。美国健康险 payment integrity 赛道，NYC，Series B 阶段：

- **做的事与鹰眼几乎逐字对应**：自研 OCR 数字化非结构化的逐项账单（itemized bills）与病历，逐行核对收费与临床记录；Autodor 平台用 AI 处理数千页病历并施加临床推理，在源头抓收费问题；明确批评"传统 payment integrity 工具依赖僵硬规则标记编码模式、缺乏对临床语境的细致理解"——和鹰眼"超越字段比对的语义稽核"是同一句话。
- **LLM + 可追溯证据链**：基于 LLM 技术，工作流完全可追溯（traceable），审核员可以对病历本身提问并**链接回原始出处**；号称比传统流程快 20 倍，因效率提升连低金额账单也能审。融资上由 FirstMark 领投 1000 万美元 A 轮，2023 年收入增长 4 倍。
- **生成式 AI 助手 Ask Autodor**：即时摘要病历、溯源临床指南、验证医疗必要性（medical necessity）——对应鹰眼的 KB2 + 取证。
- **审核维度**：编码准确性、医疗必要性、逐项账单、DRG 分类——与鹰眼 A/B/D 类规则一一对应；并强调每条改判"可辩护（defensible）"——即鹰眼的"可对质"。
- **商业模式启示**：按追回/节省金额分成（contingency-based，不收固定费用）——鹰眼可借用这个收费故事："我们只在帮基金追回钱时分成"。

**pitch 用法**："我们做的事，美国对标公司 Alaffia 已经在健康险市场验证了——LLM 读病历、证据可回链原文、比人工快 20 倍、按节省分成。中国医保基金规模更大、飞检需求刚性，这个位置目前是空的。"

### 1.4 "国家队"既是参照系也是东风（pitch 必答题：国家都有系统了你做什么？）

- 国家智能监管子系统已在**所有统筹地区**上线；2023 年通过智能审核和监控拒付、追回医保基金 **26.72 亿元**；反欺诈大数据专区已有"虚假住院""医保药品倒卖""医保电子凭证套现"等模型——虚假住院模型 2022 年查处医疗机构 62 家涉及 1.5 亿元，2023 年拉网排查追回 3.8 亿余元。
- 智能监管改革试点（医保办函〔2025〕40号）：事前提醒模块免费接入；"两库"开放共享，允许大型医疗机构本地化部署嵌入 HIS。
- **回答口径**：国家系统强在"结构化数据 + 已编码规则"的大规模筛查；留白恰好是"扫描件/照片/自由文本病历的语义级核验"——飞检现场拿到的就是这些非结构化材料。鹰眼不替代国家系统，是给稽核员配在国家系统筛出线索之后的**取证放大镜**。

## 3. AI 论文武器库（机制 → 鹰眼模块映射）

> 每条都写"落到鹰眼哪里"，不落地的不收。前两条已在04号文档（辩论类），此处补齐验证/防幻觉类。

| 论文/机制 | 一句话机制 | 落到鹰眼哪里 |
|---|---|---|
| Multiagent Debate（Du et al. 2023, arxiv 2305.14325） | 多实例互相批评后共识，提升事实性 | 控辩裁机制（已入04） |
| Tool-MAD（arxiv 2601.04742） | 辩论 agent 各配检索工具，按辩论进展迭代取证 | 控辩双方都能查 KB（已入04） |
| **CoVe 验证链**（Dhuliawala et al., ACL Findings 2024） | 起草→**规划验证问题**→**独立回答**（不受原稿污染）→修订定稿 | **取证自检环**：稽核 agent 出疑点草稿后，强制生成 3–5 个验证问题（"医嘱单里到底有无 X？""检测报告日期是否早于首次用药？"），逐题独立回查材料再定稿。这就是"反复核对"的论文版，单 agent 即可实现，黑客松性价比最高 |
| **SelfCheckGPT**（Manakul et al., EMNLP 2023） | 同问多次采样，回答发散=瞎编，一致=可信 | ① 高风险疑点温度采样 3 次独立判定，不一致→自动降级线索；② 规则准入"双盲一致性"验收（第4节）的理论依据 |
| Self-Consistency（Wang et al. 2022） | 多路推理多数表决 | 高金额疑点（如单条>1万元）走 3 路推理投票后输出 |
| **Self-RAG**（Asai et al., ICLR 2024） | 生成时自我反思：检索的内容是否真支持结论 | 取证步骤加"支持度自评"：引用的 KB 原文若不支持结论，三要素门禁不放行——把门禁从"有没有引文"升级为"引文支不支持" |
| CRAG 纠正性检索（arxiv 2401.15884） | 检索质量评估器，检索差→纠正性再检索 | KB 检索失败/低置信时的降级路径：换检索式重查→仍失败则疑点降级并注明"知识库未命中" |
| GraphRAG / RAPTOR | 跨文档全局推理 / 长文层级摘要树 | P2：画像层（跨病历全局问题）与超长病历（数百页住院记录）的摘要检索 |
| **LLM-as-Judge 偏差与缓解**（Zheng et al. 2024；2026 系统评测 arxiv 2604.23178 等） | 裁判模型有位置偏差、冗长偏差、自我偏好、迎合偏差；缓解：位置交换+不一致判平、多裁判集成、校准量表、强制 CoT | **裁判 agent 设计规范**：① 控/辩材料**交换位置二次裁决**，两次不一致→判平→降级线索；② 裁判与辩手用**不同模型**（避免自我偏好）；③ 裁决量表写死为三要素清单（校准量表）；④ 防冗长偏差：裁决看证据条目数与定位质量，不看谁话多 |
| RAGTruth（arxiv 2401.00396） | 用 RAG 幻觉语料微调小模型做幻觉检测器，可比肩 GPT-4 提示法 | 远期：积累稽核场景的幻觉标注，训练轻量"稽核幻觉检测器"做出口质检（私有化部署友好） |

**优先级**：黑客松周期内落 CoVe（取证自检）+ SelfCheckGPT（采样一致性）+ 裁判位置交换三件，成本低、对"可信度"叙事加成最大。Self-RAG 支持度自评做进三要素门禁。其余 P2。

## 4. 规则准入机制 V2：「三审三验」流水线 ⭐

> 你的直觉完全正确——规则准入应该扩展为"反复核对 + 多 agent 审核"。这里把它形式化成可执行的流水线。核心立意：**疑点有三要素门禁，规则进库也要过三审三验——可信度从规则出生那一刻就开始管。**

### 4.0 规则状态机

```
draft（起草）→ in_review（三审）→ shadow（三验/影子期）→ active（在役）
                     ↑__________________________________________|
                     re_review（误报回流触发再审） → deprecated（退役）
```

每条规则的 Schema 增加字段：`status`、`review_log[]`（三审记录）、`test_cases[]`（≥3阳性+≥3阴性）、`shadow_metrics{}`（影子运行精确率）、`false_positive_count`（在役误报计数）。

### 4.1 三审（多 agent 评审，任一审打回即退回 draft）

**审1 · 政策合规审（Policy-Auditor agent）**
- 核什么：`policy_basis` 每条引用必须能在 KB1 实际检索到原文；效力级别、版本、地区、生效区间与规则适用范围匹配；条款理解无曲解。
- 一票否决项：凭记忆写条款号/原文而 KB1 检索不到 → 直接打回（防幻觉条款进规则库）。

**审2 · 临床合理审（Clinical-Reviewer agent）**
- 核什么：用 KB2 反向提问——"什么临床情况下这个行为是**合理**的？"把所有合理情形逐条写进 `exclusions`。
- 价值：这是误报防控的源头。例：分解住院规则若没写"放化疗周期除外"，肿瘤场景必然批量误报。除外不完备 → 打回。

**审3 · 工程可执行审（Engineering-Reviewer agent）**
- 核什么：`trigger_logic` 对任意材料包都能产出三态之一（疑点/线索/不输出），不存在判不动的歧义；`evidence_min_set` 每项可定位到具体单据；阈值全部参数化未写死。
- 测法：给一份空材料包和一份标准材料包试运行，跑不出确定结果 → 重写。

### 4.2 三验（"反复核对"的形式化）

**验1 · 红蓝对抗（用例即测试）**
- 每条规则强制携带 ≥3 个阳性用例（应触发）+ ≥3 个阴性干扰用例（看似违规实则合规，**不应**触发）才能进入验收——这是模拟病历包"干扰项"思想的制度化。
- 红队 agent 专职生成边界病历攻击规则（"医嘱用了套餐名""外院检测报告夹在第 87 页"），攻破 → 补 exclusions 或修 trigger_logic，用例入库防回归。
- 工具：promptfoo 把用例当测试批量回归。

**验2 · 双盲一致性**
- 两个独立 agent 实例（不同温度，条件允许时不同模型）对同批用例**互不可见地**独立判定；一致率低于阈值（建议 90%）→ 说明 trigger_logic 写得有歧义，是规则的问题不是模型的问题 → 重写。
- 理论依据：SelfCheckGPT 的一致性思想，从"检验答案"移植到"检验规则文本质量"。

**验3 · 影子运行**
- 新规则在历史已人工复核的材料上**静默**跑（不进报告），统计精确率/误报率。
- 准入线：误报率 > 阈值（建议 10%）→ 不得转 active，或永久降级为"仅出线索"。黑客松期用模拟病历包集充当历史集。

### 4.3 在役治理（准入不是终点）

- **误报回流**：复核界面的"误报"标记累计 ≥N 次（建议 3 次）→ 自动触发 re_review。这是恒识"规则沉淀"的对偶——他们强调沉淀新规则，鹰眼同时强调**淘汰坏规则**。
- **政策失效联动**：KB1 条款 `effective_to` 到期 → 挂接该条款的规则自动 deprecated 并告警。目录年度调整时尤其关键（限定支付备注每年变）。
- **全程可审计**：每条规则携带评审记录、用例、影子指标、误报史——"**规则本身也有证据链**"，监管客户可审计，写进 pitch。

### 4.4 黑客松落地剂量

完整流水线是产品态。黑客松最小可演示版：① 给现有 41 条规则补齐 6 用例/条（红蓝用例库）；② 用 promptfoo 跑一轮全量回归出"规则质检报告"；③ 现场展示一条规则因干扰用例攻破而自动补 exclusions 的过程——评委看到的是"连规则都在被对抗测试"，可信度叙事闭环。
