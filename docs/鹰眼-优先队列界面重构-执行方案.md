# 鹰眼 · 稽核优先队列界面重构 · 执行方案

> 目标页面：`/priority.html`（稽核优先队列）
> 诊断结论：页面是按「工程师调试台」长出来的，不是按「稽核人员工作台」设计的。
> 三大病灶：① 内部字段名/工程术语裸露；② 三套分类徽章混用；③ 9 个区块堆叠、核心队列被埋没。
> 本文档给出 4 个阶段的落地步骤，每个阶段独立可交付、可验收，按顺序执行风险最小。

## 0. 范围与红线

### 0.1 涉及文件

### 0.2 红线（不许碰）

1. **不改任何后端接口和字段名**。`/api/priority/rank`、`/api/audit/batch`、`/api/priority/config` 等的请求/响应结构原样保留。改的只是「展示层怎么称呼它们」。

## 阶段 1 · 术语翻译层（纯文案，零风险，先做）

**原则：界面上只出现中文业务词。内部字段名（api_score、tier、shadow…）只允许出现在两个地方：悬停提示的括号里、导出文件里。**

### 1.1 术语对照表（本次改造的唯一词汇标准）

| 内部术语（代码不动） | 界面现状 | 改为（界面统一用词） |
|---|---|---|
| `api_score` | api_score | **优先指数** |
| `tier` / tier1 / tier2 | 层级 tier·疑点>线索 | **有疑点** / **仅线索**（不再出现 tier 字样） |
| `shadow` / 观察期桶 | 🌓 观察期桶 shadow | **新规则试运行**（试运行期间不计分） |
| `boundary` / 边界基准桶 | 🧪 边界基准桶 boundary·G0 | **零误报对照组** |
| 队首 N 入队 | 🎯 队首 N 入队 | **稽核前 N 名** |
| PII / 列表脱敏 | 列表脱敏（隐藏姓名等 PII） | **隐藏患者姓名** |
| 历史命中 患者/科室 | 100% / 81% | **患者 100% · 科室 81%**（写进单元格，不靠悬停） |
| EC / AMT / SEV | EC 证据闭环度: 0.59 | **证据完整度 / 金额权重 / 罚则严重度**（归因弹窗内） |

| E1 / E2 / E3 / G0 | 散见于按钮 title 和说明文字 | **全部删除**，这是内部迭代编号，用户不需要知道 |

## 阶段 2 · 三套徽章收敛为一个「风险档位」

### 2.1 现状问题

同一行同时出现三套语义相近、颜色撞车的徽章：

| 维度 | 取值 | 现状样式 |
|---|---|---|
| `nature`（案卷三档） | 明确违规 / 可疑 / 干净 | 红实心 / 黄 / 绿（`tri-badge`） |
| `tier`（层级） | 疑点 / 线索 | 红 / 黄（`tier-badge`） |
| `violation_nature` | 主观嫌疑 / 非主观差错 | 又是红 / 黄（`natureBadge()`） |

### 2.2 收敛规则

**表格里只保留一个主徽章**，由 `nature` 映射，全页颜色语义唯一：

| nature | 主徽章文案 | 颜色语义 |
|---|---|---|
| 明确违规 | **重点核查** | 红 = 可直接处置 |
| 可疑 | **需要合议** | 黄 = 需人工判断 |
| 干净 | **暂无异常** | 绿 = 观察 |

- `tier`（有疑点/仅线索）降级为主徽章旁的**灰色小字**，不再用彩色徽章。
- `violation_nature`（主观嫌疑/非主观差错）**从表格移除**，只在详情抽屉和归因里展示；「主观嫌疑」在详情里用红字标注即可。
- 分节行（nature-section）的徽章与提示保留，但复用同一主徽章组件，避免第二套样式。

### 2.3 实施步骤

1. `priority.js`：
   - 新增 `riskBadge(nature)` 统一函数，替换 `caseNatureBadge()` 的调用点；`tierBadge()` 改为输出 `<span class="tier-hint">有疑点</span>`（灰色小字）。
   - `renderQueue()` 第 2 列由「三档徽章 + 换行 + tier 徽章」改为「主徽章 + 小字」一行。
   - 「首要违规」列删掉 `natureBadge(row.violation_nature)`，只留违规类型文本 + risk_tags。
   - `renderFindings()`（详情内）保留 nature 徽章 + 补充 violation_nature 中文行。
2. `priority.css`：删除 `.tier-1/.tier-2/.tier-3` 彩色定义，新增 `.tier-hint{font-size:10px;color:var(--muted)}`；`tri-badge` 系列重命名语义类名（可保留旧类名做别名，避免 dashboard.js 引用断裂——先 `rg "tri-badge|tier-badge" prototype/app/public/` 确认引用面）。
3. `dashboard.js` 若引用了同款徽章类，样式同步。

### 2.4 阶段 2 验收

- 任一表格行内彩色徽章 ≤ 1 个；红色只表达「重点核查」一个含义。
- `node scripts/verify-dashboard-frontend.js` 通过。

## 阶段 3 · 信息架构重排（一屏一件事）

### 3.1 区块重排

现状顺序：词典 → 筛选 → 画像 → 操作条 → 筛查演示 → 预警台账 → **队列** → 观察桶 → 边界桶 → 详情。

目标顺序：

```
① 任务引导条（新增，一句话 + 主按钮）
② 筛选条（收成一行，含「隐藏患者姓名/院端自查用词」两个开关）
③ 稽核队列表（主体，占首屏）
④ 页签区（Tab）：历史画像 | 事前预警台账 | 新规则试运行 | 零误报对照组
⑤ 详情 → 改为右侧抽屉（不再是页底追加区块）
```

具体步骤：

1. **删除词典面板**：移除 `#glossaryMount` 区块与 `glossaryPanelHtml(false)` 调用（`glossaryPanelHtml` 函数本身保留给 dashboard）。替代：表头「优先指数」旁放一个 `?` 图标，点击弹出精简说明（复用 breakdownModal 的壳）。
2. **新增任务引导条**：`loadRank()` 成功后渲染：`当前 27 个案卷待排查：重点核查 8 · 需要合议 15 · 暂无异常 4，建议先处理前 10 名` + 主按钮「开始批量稽核」（即原 队首N入队）。数据全部来自现有 `data.queue` 的 nature 统计，无需新接口。
3. **画像 / 筛查演示 / 预警台账收进页签**：HTML 用简单的 tab 切换（无需框架，`data-tab` + `classList.toggle` 即可），默认不激活任何 tab（全部收起）；预警台账有待办时在 tab 标题上显示红点数字（`pending_supervision.length`）。
4. **空桶隐藏**：`shadow_bucket`、`boundary_bucket` 为空时对应 tab 整个隐藏（当前数据 shadow=0，即默认少一个 tab）。
5. **详情改右侧抽屉**：`#detailPanel` 样式改为 `position:fixed; right:0; top:0; height:100vh; width:min(560px, 92vw)`，滑入动画 + 遮罩点击关闭；删除 `scrollIntoView` 逻辑。点表格行任意空白处即打开抽屉（不只靠「详情」按钮）。

### 3.2 表格瘦身：11 列 → 7 列

| # | 列 | 内容 |
|---|---|---|
| 1 | 选择框 | 不变 |
| 2 | 排名 | 新增：#1 #2 #3…（比裸分数直观；分数进第 3 列） |
| 3 | 风险档位 | 主徽章 + tier 小字（阶段 2 产物）+ 优先指数小字与横条 |
| 4 | 案卷 | 标题 + 灰色 case_id + 科室（科室列并入此列） |
| 5 | 涉及金额 | 金额 + 灰色小字 `疑点2/线索0` |
| 6 | 主要问题与建议 | 违规类型 + risk_tags + 处置建议（原两列合并；处置建议截断，全文进抽屉） |
| 7 | 操作 | 主按钮「详情」，「归因/举证包」收进 `⋯` 下拉 |

优先指数横条：`width = row.api_score / queue最大分 × 100%`，纯前端计算，让 178.6 这种无量纲数字有了参照。

### 3.3 归因弹窗结构化

`formatBreakdownText` 的纯文本 `<pre>` 改为 HTML 结构（`priority-glossary.js` 新增 `formatBreakdownHtml(row)`，`formatBreakdownText` 保留给 dashboard 兜底）：

- 顶部一行乘法算式：`基础分 76 × 历史 1.5 × 多规则 1.3 × 离群 1.2 = 178.6`；
- 下面每个因子一行：中文名 + 数值 + 一句解释 + 相对贡献横条；
- 深色代码块样式（`.pri-pre`）不再用于此弹窗。

### 3.4 违规统计表去 JSON 化

`previewReport()` 现在把 `JSON.stringify` 直接打给用户。改为：把 `d.tables.recognition` / `d.tables.amount` 渲染成两张 `pri-table` HTML 表格；「导出 Markdown」按钮行为不变。

### 3.5 阶段 3 验收

- 首屏（1440×900）能看到：任务引导条 + 筛选条 + 队列表前 5 行。
- 表格列数 = 7；点行打开右侧抽屉；归因弹窗无 `<pre>` 代码块观感；统计表预览无裸 JSON。
- `node scripts/verify-dashboard-frontend.js` 通过。
