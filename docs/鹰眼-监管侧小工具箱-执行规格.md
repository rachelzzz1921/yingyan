# 鹰眼 · 监管侧小工具箱 · 执行规格

> **用途**：发给 Codex / 执行者，按本文落地监管侧小工具。  
> **版本**：2026-07-03  
> **背景**：大平台（工作台、优先通路、黄金闭环）能力完整，但监管人员不一定愿意打开第三方大平台。要把能力拆成**小工具**，嵌进他们已有的 Excel、线索台账、飞检文件夹、专家复核流程。  
> **原则**：同一个引擎、同一套 KB、同一条 G0 零误报红线；**换输入方式 + 换输出模板 = 一个新工具**，不重写引擎。

---

## 1. 一句话目标

做一个 **监管侧小工具箱**：

- **入口**：桌面哨兵（CLI）+ 网页工具菜单（`regulator-plugins.html`）
- **精做 3 个工具**（演示主线）：疑点分诊器、规则阈值试算器、飞检行装包生成器
- **菜单预览 4 个工具**（讲扩展性）：追溯码即查、申诉批阅助手、数据体检哨兵、判例速查卡
- **保留原「三件套」工作流叙事**，但与上面合并去重，不重复做页面

## 5. 精做工具规格（P0）

### 5.1 ① 疑点分诊器

**解决什么**：智能监管系统导出 1 万条疑点，经办/监管要过好几遍，第一遍粗筛最痛。

**给谁用**：监管处、线索核查员、经办审核负责人。

**输入**：

- Excel / CSV 疑点清单（粘贴或上传）
- 或演示：1000 条固定集 `prototype/data/screening/settlements_1000.json`

**输出三档**（措辞禁止写「拒付/放行」，避免越界）：

| 档位 | 展示名 | 含义 |
|---|---|---|
| A | **建议重点核查** | 金额/严重度高，证据较完整 |
| B | **建议补材料后核查** | 有风险但证据缺口明显 |
| C | **建议暂缓 / 观察** | 弱线索、低金额、高误报风险 |

每条还要带：

- 命中规则 ID + 名称
- 金额
- **分档依据**（1 句话：为什么排在这个档）
- **可能抗辩点**（不叫「申诉成立概率」——太像 AI 裁决）
- **建议动作**：优先核查 / 调阅 XX 材料 / 暂缓

**已有可复用代码**：

| 模块 | 路径 |
|---|---|
| 批量筛查引擎 | `prototype/app/engine/screening.js` |
| 1000 条数据 | `prototype/data/screening/settlements_1000.json` |
| 优先队列评分 | `prototype/app/engine/priority-score.js`、`priority-nature.js` |
| 批量 API | `POST /api/screening/run`、`POST /api/screening/rows` |
| 优先队列页 | `prototype/app/public/priority.html` |
| 剪贴板哨兵 | `plugin/desktop-sentinel/clipboard-sentinel.js` → `/api/screening/rows` |

**要新建/改造**：

1. 新建 `prototype/app/public/regulator-triage.html`（或在 `priority.html` 顶部加「监管 · 疑点分诊模式」banner，二选一，**推荐新建轻量页**，避免改乱 priority 现有布局）
2. 页面第一屏写清：给谁用、输入什么、输出三档
3. 按钮：**粘贴 Excel 演示** / **跑 1000 条演示** / **打开 Top1 飞检行装包**
4. 桌面哨兵：`clipboard-sentinel.js` 输出文案对齐三档命名（现在是 🟥/🟨/🟩，可保留 emoji，但加文字档名）

**验收**：

- [ ] 30 秒内能演示：1000 条 → 三档漏斗 → Top 10 列表
- [ ] 每条有「分档依据」字段，不是黑盒排序
- [ ] 剪贴板粘贴 TSV 也能跑（复用现有 screening/rows）
- [ ] 口播可用：「第一遍初审的粗筛就是它干的」

### 5.2 ② 规则阈值试算器

**解决什么**：国家政策把阈值、参数、应用场景下放地方自定；地方缺「调一下看影响」的工具。

**给谁用**：地方医保局规则管理员、信息科、飞检筹备组。

**输入**：

- 选择一条规则（下拉，演示默认 `AGE-101` 或 `QTY-901`）
- 拖动阈值滑块（如：年龄上限、数量上限、比例阈值）

**输出**（即时刷新）：

- 触发量变化（+N / -N 条）
- 在 1000 条或历史案卷上的命中 diff
- **疑似误报压力**（干净案卷新增命中数，必须为 0 才标绿）
- 简短结论：「阈值收紧 → 触发减少 X，误报压力 Y」

**已有可复用代码**：

| 模块 | 路径 |
|---|---|
| 历史案卷回放 | `prototype/app/engine/precip-replay.js`（`replayDraft`） |
| 规则 overlay | `prototype/app/engine/rule-precipitation-service.js` |
| 回放 API | server.js 里 precip replay 相关路由（搜 `replayDraft`） |
| 机构监测阈值 | `prototype/app/engine/stats-monitoring.js`（`thresholdFor`） |

**要新建**：

1. 新建 `prototype/app/public/regulator-threshold.html`
2. 新建轻量 API 或复用 replay：`POST /api/rules/threshold-trial`  
   - body: `{ rule_id, param_key, param_value }`  
   - 返回: `{ baseline_hits, trial_hits, delta, new_false_positives, cases_changed[] }`
3. 第一版可只对 **1～2 条规则**做真滑块（AGE-101 年龄、QTY-901 数量），其余规则下拉灰掉标「演示扩展」

**实现提示**（给 Codex）：

- 不要重写引擎；用 `applyOverlaysToRules` 临时 patch 规则 params，再对 `settlements_1000.json` + 若干 `case_*` 跑 `screenExternalRows` 或 `runAudit`
- 干净案卷（`embedded_violation_count === 0`）新增命中 = 误报压力，必须单独计数
- UI：左侧滑块，右侧数字卡片 + 简单 before/after 表格

**验收**：

- [ ] 拖滑块 2 秒内看到数字变化
- [ ] 干净案卷误报增量可见，且 G0 相关案卷不为红
- [ ] 口播：「国家给地方的是规则自主权，鹰眼给的是试算工具」

### 5.3 ③ 飞检行装包生成器

**解决什么**：飞检进场前需要标准化材料——疑点清单、证据索引、现场核查任务单。专家原话：「我们需要用标准化的材料去执行我们的任务」。

**给谁用**：飞检组长、现场检查员、第三方 IT 协查。

**输入**：

- 机构名 + 期间（演示可固定）
- 或：从疑点分诊 Top1 一键带入
- 或：选择案卷 `main` / 上传材料包

**输出一个「行装包」**（HTML 可打印 / Markdown 下载）：

1. **疑点清单**（规则、金额、三档性质）
2. **证据索引**（费用行 / 病历锚点 / 政策 ref_id）
3. **现场核查任务单**（查什么、问什么、调阅什么）
4. **缺失材料清单**
5. （可选）**询问笔录要点**（不做最终笔录，只列建议问题）

**已有可复用代码**：

| 模块 | 路径 |
|---|---|
| 单案稽核 | `POST /api/audit` |
| 举证包 | `POST /api/evidence-package` → `engine/evidence-package.js` |
| 领导版报告 | `engine/leader-report.js` |
| 材料导入 | `/intake.html` |
| 黄金闭环案卷快读 | `/golden.html` 步骤 1–2 |
| 桌面文件夹哨兵 | `plugin/desktop-sentinel/sentinel.js` |

**要新建**：

1. 新建 `prototype/app/public/regulator-fieldkit.html`（页面标题：**飞检行装包生成器**）
2. 新建 `prototype/app/engine/field-kit.js`：聚合 audit 报告 + evidence-package + 任务单模板
3. 可选 API：`POST /api/regulator/field-kit` `{ case_id | org_id, period }` → HTML/MD

**页面布局**：

- 左：选案卷 / 从分诊带入
- 右：行装包预览（可折叠五块）
- 底部：**下载 Markdown** / **打印** / **对争议项进入复核**（链 `agent-chain.html?case_id=`）

**验收**：

- [ ] 选 `main` 一键生成完整行装包
- [ ] 含 T-201 证据索引 + 调阅「分子病理报告」建议
- [ ] 从疑点分诊 Top1 能跳转带入
- [ ] 口播旁可放专家原话（文档内引用，不编造）

## 6. 预览工具规格（P2 · 菜单可见即可）

### 6.1 ④ 追溯码即查

- **输入**：粘贴追溯码
- **规则**：TRACE-101 回流药 / 重复结算
- **复用**：`screening.js` 行级筛查 + `case_pharmacy` 埋点
- **交付**：菜单项 + 简单输入框页或哨兵 `--stdin` 模式；**半天量级**
- **验收**：输入演示追溯码 → 秒出是否重复 / 命中说明

### 6.2 ⑤ 申诉批阅助手

- **定位**：医院申诉材料进来 → 控辩裁参考意见（**不是最终裁定**）
- **复用**：`agent-chain.html` + `POST /api/debate`
- **交付**：菜单链到 agent-chain，标题改为「申诉批阅助手（预览）」；默认对抗分诊开
- **验收**：T-205 演示降级为线索

### 6.3 ⑥ 数据体检哨兵

- **输入**：拖入医院报送 Excel
- **输出**：字段缺失、编码格式、孤儿行
- **交付**：菜单项 + 最小校验（列名检测 + 空值统计）；**一下午量级**
- **验收**：拖 demo xlsx → 出 3 类问题列表

### 6.4 ⑦ 判例速查卡

- **输入**：违规类型 + 金额区间
- **输出**：`prototype/data/kb/exposure_cases.json` 同类公开案例摘要
- **措辞红线**：
  - ✅ 「公开案例参照」「历史处理区间」
  - ❌ 「裁量建议」「建议罚款倍数」「AI 处罚建议」
- **验收**：搜「回流药」→ 出同济/无锡等已核实条目链接
