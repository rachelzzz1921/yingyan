# 鹰眼插件 · 三角色完整闭环 · 演示脚本

> 一句话:**同一个事前哨兵,盯住医生开单 / 编码员编码 / 结算员结算三个时点;每个角色都能"命中→整改翻绿放行 或 坚持提交转监管";未遵从的单进监管重点审核,监管回执再写回院端看板。** 违规消灭在萌芽,消灭不掉的进监管。
>
> 演示前:`cd prototype/app && node server.js`(:3700);演示前点看板"清空"或重启服务(启动自动清陈旧,只留今日)。

## 主线脚本(≈4 分钟,投屏)

### ① 医生开单(30")
1. `/mockhis.html`,场景下拉选 **S1 未成年+喹诺酮**(16 岁开左氧氟沙星)。
2. 点 **🦅 鹰眼审方** → 浮层弹 **明确违规 AGE-101**(两库 14/18 岁分层依据 + 政策原文,可折叠展开)。
3. **演"整改翻绿"**:点 **✓ 采纳建议·整改** → 开单表里那两行喹诺酮/儿童制剂**当场消失** → **自动重新审方 → 翻绿"已整改·重新审方通过·可提交"**。
   > 话术:"违规消灭在萌芽——医生当场改了,监管侧根本不用处理这一条。"
4. (对照)再选 **S2 女性开男性专属项目(PSA)** → 命中 F-001 → 这次点 **⚠ 坚持提交** → 选理由"临床确有必要" → **"已提醒·未遵从·转监管重点审核"**。
   > 话术:"消灭不掉的,不放过——带着医生填的理由,进监管重点审核。"

### ② 编码员(40")
1. `/coder-station.html`,场景 **C1 重症肺炎·病历无重症依据(高套)**。
2. 点 **🦅 提交编码前校验** → 弹 **可疑 D-401 DRG 高套**:
   - 主诊断按重症编码 → 入 **ES31(权重1.28)**;病历无重症依据 → 应入 **ES35(权重0.76)**;
   - **实施细则第三十五条(一)"两病种支付标准差额" = (1.28−0.76)×¥8500 = ¥4,420**(公式 + 真条文,标注"演示子集·费率虚构")。
3. 点 **✓ 采纳建议·降编到 ES35** → 主诊断降编 → **自动重校验 → 翻绿"可提交编码"**。
   > 话术:"高套从事后翻账,前移到编码桌当场拦——把 D-401 做成事前。"
4. (可选)其它场景点坚持提交,转监管。

### ③ 结算员(30")
1. `/settle-station.html`(16 岁患者出院费用清单 6 行,埋 3 处)。
2. 点 **🦅 结算前整单自查** → 漏斗:**明确违规 2(AGE-101/TRACE-101)· 可疑 1(QTY-901)· 放行 3**。
   - TRACE-101:恩替卡韦同一追溯码 6-11/6-16 两次结算(回流药,仿国家局 2026-05 曝光)。
3. 点 **✓ 采纳建议·整改标注行** → 命中的 3 行**从费用清单剔除** → **重校验通过 → 可结算**。
   > 话术:"结算员一键把整份清单过一遍,与监管侧一千条是同一引擎——第一遍漏斗在院端就跑了。"

### ④ 院端运营看板(40")
`/plugin-dashboard.html`,一屏讲完院端战果:
- 顶部 5 卡:**今日提醒 / 遵从率 / 萌芽拦截 / 未遵从待监管 / 监管已回执**。
- **三角色触点分布 + 逐角色遵从率**(医生/编码员/结算员各自采纳率)。
- **未遵从理由 → 规则优化信号**:同规则被同类理由未遵从≥2 次 → "建议接入外院检测数据源,减少误提醒"(反哺规则库)。
- 未遵从清单(带角色列)。
  > 话术给葛文婧/行知:"报告领导要的就是这一屏——拦了多少、谁在坚持、哪条规则要优化。"

### ⑤ 监管工作台 · 回执闭环(40")
`/priority.html` 顶部 **⚠ 事前已提醒·未遵从 · 重点审核预警台账**:
1. 列出三角色的未遵从单(角色/科室/患者/命中/坚持理由)。
2. 对一条点 **核实违规**(或驳回误报/已联系院端)→ 该行变 **"监管回执:核实违规"** 徽标。
3. 切回 `/plugin-dashboard.html` → **"监管已回执"卡 +1**。
   > 话术:"院端推给监管、监管核完回执写回院端——同一台账,双向闭环,不是单向甩单。"

## 一句话收尾(游园会 30 秒钩子版)
"三个角色——医生、编码员、结算员——被同一个哨兵在开单、编码、结算三个时点盯着;能当场改的当场消灭在萌芽,改不掉的带着理由进监管重点审核,监管核完回执再写回院端看板。违规,要么萌芽消灭,要么监管兜底,没有中间地带。"

## 诚实边界(被追问时的口径)
- **整改"翻绿"不是删数据蒙混**:医生=撤销该医嘱/改开;编码=按实际应入组降编(更正病案首页,非篡改);结算=剔除/退费该行。演示里的"消失"对应真实世界的"撤单/更正/退费"。
- **监管回执**:当前登记核实/驳回状态(双向闭环的最小闭合);接入正式处罚/申诉流程为下一步。
- **规则优化信号**:只是候选提示,仍需人工采纳,不自动改规则(对齐"成熟一条、应用一条")。
- **DRG 分组**:演示子集数据表(CHS-DRG 2.0,肺炎 J12–J18 示范病种),费率为虚构脱敏值;vendor OpenDRG 到位即切全量。
- **数据不出机**:插件/哨兵只与本地引擎通信;剪贴板通道只发解析后的结构化医保字段。

## 降级预案(现场)
- 引擎未连接:浮层/工作站显示红色"校验未完成:引擎未连接",不伪装成绿色放行。
- 台账脏数据:启动自动清陈旧;或看板"清空"(演示态放行,配了 token 的环境需令牌)。
- 剪贴板 pbpaste 不可用:`pbpaste | node plugin/desktop-sentinel/clipboard-sentinel.js --stdin` 回退。
