# 1. 项目定义与赛道

**项目名**：鹰眼 —— 医保基金稽核智能体
**赛道**：医保基金监管领域

**痛点**：医保基金违规行为隐藏在海量非结构化材料中，而稽核依赖人工。一次飞行检查中，稽核人员需在数日内审阅一家医院成千上万份病历、费用清单与处方，抽查覆盖率低、漏检率高、高度依赖个人经验。近年国家医保局飞检追回基金规模屡创新高，恰恰说明存量违规远未被充分发现。

**方案**：为稽核员打造"AI 初筛员"。输入患者完整就诊材料包（费用清单、病历、医嘱、检查报告，支持扫描件/照片/PDF），Agent 自动完成"费用—诊断—医嘱"三方交叉验证，输出结构化稽核报告：疑点按风险分级，每个疑点附带三要素——原始证据定位（具体到费用行与病历原文）、违反的政策条款原文（医保目录限定支付范围、诊疗规范）、完整推理过程。可识别串换收费、超标准收费、无指征用药、分解住院等典型违规。

**技术路线**：多模态文档解析（原生理解扫描件与表格）→ 医保目录与支付规则 RAG 知识库 → Agent 循环稽核推理与取证。无需模型训练，可私有化部署，保障数据不出域。

**创新点**：
1. 面向非结构化病历的语义级稽核，超越传统规则引擎的字段比对；
2. 强制证据链输出——引不出原文的疑点不输出，宁漏报不误报，结论可直接用于与医疗机构对质；
3. 单份材料人工审阅约 40 分钟，Agent 约 90 秒，将人力从初筛释放到复核与查处。

**黑客松交付**：可运行原型 + 含预埋违规点的模拟病历包现场演示。

**已确认的关键决策（2026-06-12）**：
- Demo 主场景：**肿瘤**（非小细胞肺癌 NSCLC 主线 + 结直肠癌副线）
- 锚定范围：**全国 + 江苏省**（地方价格目录与负面清单以覆盖层挂接）
- 支付方式违规：**轻量纳入**（保留高套分组 D-401、费用转嫁 D-402，其余以按项目付费为主）
- 风险分级：采用**疑点（证据闭环）/ 线索（证据不全）/ 不输出**三态，对齐官方"明确违规/可疑"口径

## 2.2 国家"两库"框架（KB1 第三层 + 规则分层的直接依据）

**核心情报**（鹰眼规则分层与命名直接对齐它）：
- 2022年3月《医疗保障基金智能审核和监控知识库、规则库管理办法（试行）》（医保发〔2022〕12号）明确了"两库"功能定位、建设原则、建设程序及各级医保部门职责。知识库是智能审核和监控所需知识和依据的集合；规则库是基于知识库判断监管对象行为合法合规合理性的逻辑、参数指标、参考阈值及判断等级的集合。
- 2023-05-15 发布《"两库"框架体系（1.0版）》。**知识库**含政策类、管理类、医疗类三类目录（法律法规、医保信息业务编码、医药学知识、管理规范）。**规则库含 79 条规则**：
  - 一级分类：政策类 30 条、管理类 28 条、医疗类 21 条
  - 政策类二级：药品政策限定类 12、医疗服务项目政策限定类 15、医用耗材政策限定类 3
  - 管理类二级：信息数据监管类 8、药品监管类 4、医用耗材监管类 1、行为主体监管类 7、统计指标监测类 8
  - 医疗类二级：药品合理使用类 12、医疗服务项目合理使用类 6、医用耗材合理使用类 3
- **运行逻辑（鹰眼三态输出的官方依据）**：违反"政策限定类"规则的运行结果是"明确违规"，可自动拦截；违反"合理使用类"规则的结果多是"可疑"，需充分听取医药机构申诉意见后确定是否支付。"两库"实行动态调整，原则上每年一次年度调整。
- 来源：nhsa.gov.cn 2023-05-15 公告页（art_109_10611.html / art_105_10612.html）及附件 PDF。

- 只有经病理诊断结果确诊的恶性肿瘤，才可使用抗肿瘤药物；单纯依据临床症状、体征和影像学结果得出临床诊断的肿瘤患者，没有抗肿瘤药物治疗的指征（难以获取病理的如胰腺癌可参照指南执行）。→ T-202
- 对于明确作用靶点的药物，须遵循靶点检测后方可使用的原则；检测方法应经 NMPA 批准（特别是伴随诊断验证方法）；不得在未做相关检测的情况下盲目用药。→ T-201
- 特殊情况下超说明书用药：使用权仅限三级医院授权的高级专业技术职称医师，遵循患者知情同意原则，做好用药监测；循证依据采纳顺序为其他国家/地区说明书用法 → 国际权威学协会诊疗规范/指南 → 国家级学协会经卫健委认可的诊疗规范/指南/临床路径。→ T-203

# 3. 审核规则库雏形（通用 33 条）

## 3.1 设计原则（与产品承诺逐条对齐）

1. **证据链强制（三要素门禁）**：每条规则定义"最小证据集"。Agent 输出任何疑点前自检三要素——① 原始证据定位（费用行号、病历页码/段落原文）② 违反的政策条款原文（KB 引用 ID + 原文）③ 完整推理过程。任一缺失则降级为"线索"或不输出。**宁漏报不误报**。
2. **三态输出**：`疑点`（证据闭环，可直接对质）/ `线索`（模式异常但材料包内无法闭环，附"需调阅材料清单"）/ `不输出`（依赖未证实假设或无法定位原文）。
3. **分层执行（对齐两库运行逻辑）**：
   - **L1 确定性规则**：字段与逻辑校验，纯代码可实现（性别、时间、数量、互斥），是底线也是 LLM 输出的校验锚点。
   - **L2 语义规则**：必须读懂病历自由文本才能判定（核心创新，超越字段比对）。
   - **L3 线索规则**：单份材料包不足以定论，需跨患者/跨病历或外部数据佐证，只出线索。
4. **术语与官方对齐**：违规类型命名与《条例》第38/40条及国家局问题清单一致，报告可直接落到条款。
5. **参数化与本地化**：所有阈值（分解住院间隔、出院带药天数、护理时长上限等）做成可配置参数；地方价格目录、地方化清单以"覆盖层"挂接，不写死。

# 4. 两大知识库建设方案

## 4.1 总体架构

- **KB1 回答"违不违规"**（政策、目录、规则、负面清单）；**KB2 回答"该不该做"**（入院指征、用药指征、诊疗规范）。两库职责分离，Agent 推理时分别取证。
- 检索策略：KB1 以**结构化精确召回**为主（目录按药品行、法规按条款、清单按问题项），保证引得出原文；KB2 以**病种/药品为键的向量检索**为主。
- **引用 ID 规范**（强制证据链的工程基础）：`[KB1-文档代号-条款定位]`，如 `[KB1-条例-第38条(三)]`、`[KB1-目录2025-西药-XA01-备注]`。报告里每条政策依据必须落到引用 ID + 原文，否则三要素门禁不放行。

## 4.2 KB1 五层结构

1. **法律法规**（P0）：条例、实施细则(7号令)、社会保险法、定点管理办法 —— 按"条"分块。
2. **目录与支付范围**（P0）：2025版药品目录（"备注"列整列入库）、商保创新药目录、医保编码、立项指南、江苏价格目录 —— 按"药品/项目行"分块。
3. **智能审核规则框架**（P0）：两库管理办法、两库1.0框架（79条规则+释义）、江苏负面清单。
4. **典型问题清单**（P0）：2024/2025/2026 三版 —— 按"问题项"分块。
5. **案例与通报**（P1）：曝光台、各省典型案例 —— 抽"行为描述→定性→处理依据"三元组做 few-shot。

```json
 "violation_tags":["重复收费"],"linked_rules":["A-101"]}
```
分块原则：法规按条、目录按药品/项目行、问题清单按问题项、案例按案件。**禁止入库改写版**（改写只放规则的 trigger_logic）。

## 4.3 KB2 临床指南库 —— 按"病种包"组织

不要全量建库。每个 demo 病种建一个病种包：
```

       + 该病种高频药品说明书摘要（适应症、用法用量）
       + 对应医保目录限定支付条款（与 KB1 行级互链）
       + 该专科问题清单条目（与 KB1 互链）
```

**入库 Schema（KB2）**：
```json

 "disease_icd":"J18.x","drug_generic":null,"key_elements":["氧合指数","并发症"],
 "source":"国家卫健委临床路径(2019年版)","version_date":"2019-12","text":"要点原文/忠实摘录"}
```

## 4.4 抓取顺序与合规

**抓取优先级**：
- P0（第1周）：条例+实施细则全文；2025版药品目录"备注"列；2026问题清单（demo科室）；两库1.0附件；抗肿瘤指导原则2025版。
- P1（第2周）：江苏价格目录与负面清单；立项指南相关分册；曝光台案例。
- P2（演示前）：耗材代码、DRG/DIP 2.0 分组方案。

**合规要点**：
1. 官方附件优先（PDF/Excel 可结构化）；官微长图需 OCR + 人工校对（措辞直接进报告，错字不可接受）。
2. 政府法规公文可全文入库；**学会指南/教材有版权**，做条目级摘引保留出处，内部使用。
3. 版本治理：每条知识带 `effective_from/to`，Agent 按**就诊日期**选版本（稽核历史行为不能用新规则套旧行为）。
4. 数据不出域：知识库均为公开文件可随包分发；患者病历仅本地处理，物理分离。

# 5. 肿瘤 Demo 专科规则包

## 5.1 为什么选肿瘤（pitch 逻辑）

1. **金额大、监管热**：肿瘤是 2025 年新纳入自查自纠的重点领域，靶向药/免疫药单价高，一个疑点动辄上万元。
2. **最能体现"语义级稽核"**：靶向药能不能用取决于病理诊断、基因检测报告、治疗线数——全藏在非结构化病历里，传统规则引擎完全做不了。
3. **政策依据层层咬合**：医保目录限定支付（医保口径）+ 卫健委指导原则（诊疗口径）+ 药品说明书（法定依据）三层互锁。

## 5.2 KB2 肿瘤病种包（起步两个）

- **主线 NSCLC（非小细胞肺癌）**：靶向药生态最完整。
- **副线 结直肠癌**：RAS/BRAF 检测 + 贝伐珠单抗/西妥昔单抗，覆盖另一类检测逻辑。

**NSCLC 用药指征卡（示意 · 入库前以2025版指导原则+说明书逐条校准）**：
| 药品（类） | 必检靶点/前提 | 适应症要点 | 核对证据 |
|---|---|---|---|
| EGFR-TKI（吉非替尼/奥希替尼等） | EGFR敏感突变阳性 | 阳性方可用；三代多用于特定突变或耐药后T790M | 基因检测(EGFR)+病理 |
| ALK抑制剂（克唑替尼/阿来替尼等） | ALK融合阳性 | 阳性方可用 | ALK检测报告 |
| 抗血管生成（贝伐珠单抗等） | 多数无需靶点检测 | 联合化疗，注意禁忌 | 病理+禁忌核对 |
| 免疫检查点抑制剂（PD-1/PD-L1） | 部分需PD-L1或仅需病理 | 适应症与线数依说明书 | 病理+（如需）PD-L1+线数 |

## 5.3 肿瘤专科规则（8条 + 复用通用规则）

| ID | 名称 | 层/风险 | 触发要点 | 违规类型 | 演示价值 |

**复用的通用规则（肿瘤场景同样跑）**：
- A-108 虚记费用（化疗护理、输液类高发）
- A-105 护理等级与实际不符（化疗期护理等级虚高）
- C-301 分解住院 —— **除外放化疗按周期规律住院**（参数白名单，肿瘤场景必设例外，避免误报）
- C-302 挂床住院（周期间歇期挂床）
- B-201 限定支付通用版（肿瘤外支持治疗用药如人血白蛋白）

**T-201 详解（首推演示规则）**：
- 触发：医保结算靶向药属"明确作用靶点"类（命中 KB2 必检清单）→ 检索对应靶点检测报告 → 无报告或靶点阴性/野生型 → 疑点。
- 三态分流：无检测报告且药已用→疑点（违反"不得未做检测盲目用药"）；报告阴性仍用→疑点（无适应症）；病历提示"外院已检测"但无报告→线索（列调阅清单）。
- 最小证据：靶向药费用行 + 病理诊断原文 + 基因检测报告（或缺失说明）+ 指导原则/说明书引用 + 目录备注引用。
- 除外：无需检测的抗血管生成类后线用药；说明书未要求检测的药品。
- KB引用：`[KB2-指导原则2025-靶点检测原则]`、`[KB2-说明书-XX]`、`[KB1-目录2025-西药-XX-备注]`。

# 6. 技术路线与架构

```
输入：患者完整就诊材料包（费用清单/病历/医嘱/检查报告，扫描件/照片/PDF）
  ↓
① 多模态文档解析层：原生理解扫描件与表格，结构化为 {费用行, 医嘱项, 病程文本, 检验/病理报告, 病案首页}
  ↓
② RAG 知识库层：KB1（医保目录+支付规则+问题清单）+ KB2（临床指南+说明书+病种包）
  ↓
③ Agent 稽核推理循环：
   - 跑 F 类 L1 确定性规则（代码）
   - 按验证轴跑 L2 语义规则（LLM 读病历）
   - 命中疑点 → 强制取证（回查原文定位）→ 三要素门禁
   - L1 校验 L2 输出，冲突标记复核
  ↓
④ 输出：结构化稽核报告（疑点按风险分级，每疑点附三要素 JSON）
```

**关键工程约定**：
- 知识库检索返回 `{ref_id, 原文, 定位, 效力级别, 生效区间}`，缺任一字段视为检索失败。
- 报告政策依据仅允许引用 KB 返回的原文，**禁止 LLM 凭记忆生成条款文本**（防幻觉条款）。
- KB1 与 KB2 交叉互链（限定支付条款↔说明书↔问题清单条目），供 Agent 一跳取齐证据。
- 无需模型训练，可私有化部署，数据不出域。

## 6.1 已落地的原型架构（v1.2，可运行）

```
鹰眼原型 /Desktop/医疗黑客松/鹰眼原型/
  data/  case_NSCLC/(病历包+金标准) · rules/(41条yaml→json) · kb/(KB1政策+KB2临床)
  app/   server.js(零依赖HTTP+API) · engine/audit-engine.js(确定性规则引擎,真·计算)
         engine/debate.js(控辩裁) · engine/llm-audit.js(LLM语义路径,可选)
         public/(稽核工作台UI: 材料包↔报告 双栏, 三要素证据卡, 控辩裁, 误报防控)
  启动：cd app && node server.js → http://localhost:3700
```
- **三态可证伪**：确定性引擎对"结构化可判定"规则做真计算（F-003时间/A-109数量/A-105护理等级/T-201靶点/T-207转嫁/T-205限定），改材料包则结论随之变化；纯语义规则走 LLM 路径（`?mode=llm`，配 ANTHROPIC_API_KEY）。
- **证据链可视**：每条疑点展开即见三要素①证据定位(可点击跳材料行)②条款原文(KB1/KB2引用ID+核验状态✅/⚠)③推理过程；政策原文一律取自 KB，禁止凭记忆生成。

## 6.3 控辩裁多Agent对质机制（pitch 杀手锏，已落地原型）

把稽核现场的真实博弈搬进推理：**稽核Agent(控方)→申诉Agent(辩方)→裁判Agent(裁判)**。

```
① 稽核Agent(控方)：跑规则库，提疑点，从材料包取证，给初步定性+条款引用
② 申诉Agent(辩方)：为机构辩护——查除外情形、找反向证据、质疑证据链完整性  ← 本质=误报过滤器
③ 裁判Agent(裁判)：核验三要素是否齐备，裁定 疑点(闭环)/线索(存疑)/撤销(误报)
```
- **机制要点（直接映射产品原则）**：辩论只对高风险L2疑点/争议线索启动（**L1确定性命中不辩**，省成本/防过度生成放大幻觉）；**轮数硬上限=2轮**，无果裁判定夺；裁判标准=三要素门禁，辩方指出任一要素缺失或有效反向证据→降级线索/撤销，**这正是"宁漏报不误报"的工程实现**。
- **演示价值**：病历包 #6（T-205 升白针）现场演示——控方主张超限定支付疑点，辩方指出"前次是否重度中性粒减少"的决定性证据(前次血常规)不在本单份材料包内、证据未闭环，裁判据此**疑点→线索降级**。竞品讲"风险分级"，鹰眼能现场展示"控辩裁三方对质"，稽核员拿到的是已过对抗检验的疑点。
- 落地：黑客松阶段已用确定性脚本版（演示稳定）；进阶可用单模型多角色 prompt（已在 llm-audit 预留）或 LangGraph 真多Agent（精确控2轮+裁判介入）。

## 6.4 竞品差异化（恒识 Evercog · 同赛道）

- **借鉴补齐**：抄其"规则沉淀闭环"（稽核员复核标注→反向沉淀规则/调参数，把案例库提到工作流主干）与"多维度风险画像层"（机构/医生/科室/药品/病种，鹰眼L3线索规则的数据底座，架构图预留接口）。
- **守住优势**（它没有的）：①**机制化可信度**——"引不出原文不输出+宁漏报不误报+三要素门禁+控辩裁"，对质场景下机制承诺比能力承诺更打动人；②**规则落地深度**——41条带证据集的规则+病历包预埋点，框架谁都会画、规则写不写得出是另一回事。
- pitch 策略：框架对等之后，比的是谁真能跑出可对质的疑点。
