## 一、项目简介

**一句话**:鹰眼是一套医保基金审核引擎,按国家智能审核监控规程的七个法定环节(数据采集传输 → 数据比对 → 违规筛查 → 明细审核 → 调查核实 → 违规处理 → 评估分析)将 Agent 逐一对位编排,同一引擎复用在两条产品线:**院端(事前/自查)** 与 **监管侧(事后全覆盖)**。

**用户与场景(30 秒版)**:基金监管科审核员老李,每月面对 1 万+条疑似违规数据、每条要过 3 遍(初审/申诉复审/重审)。鹰眼把这 1 万条按**明确违规/可疑/干净**三档分诊:明确违规档带三要素证据链直接进处置;可疑档进对抗合议预演申诉;干净档放行。老李只复核真争议——一个人处理多案卷。

**技术身份**:确定性引擎(L1,毫秒级,对应"明确违规"自动拦截)+ LLM 多 Agent 合议(L2,对应"可疑"需听申诉)。这个分层与国家两库规则的"政策限定类/合理使用类"二分**同构**。

## 三、AI 创新性

### 3.1 七环节 Agent 对位法定规程(不可替代性论证)

编排拓扑不是我们发明的——它是部颁智能审核工作规程的七个法定环节的 Agent 化。**不可替代的不是 Agent 拓扑,而是每个 Agent 的 harness 里装的法定规程、两库条目和判例。** 通用模型没有这些,产不出可溯源的结论。

【架构图:七环节 → Agent 映射】

### 3.2 对抗合议三人格(信息不对称设计)

同一底模,差异来自三个杠杆:**上下文分区、弹药库分区、输出立场约束**——

| 人格 | 只能看到 | 输出契约 |
|---|---|---|
| 医院辩护 | 病历/医嘱/检验(看不到规则引擎推理) | 穷尽辩护,每辩点引用病历证据位置 |
| 监管指控 | 规则命中+两库条目+结算数据(看不到病历叙事) | 指控书格式,每指控挂法条/两库条目 ID |
| 专家裁定 | **只看两份陈述书**+两库/判例库/裁量依据(不看原始数据) | {成立/不成立/部分成立/证据不足转人工}+评分 0-100+依据链 |

三重防线:**钢人条款**(裁定前必须复述双方最强论点)、**引用硬校验**(依据必须是可解析 KB 条目 ID,引用不出即代码层强制转人工,不靠模型自觉)、**评分制**(对齐申诉评定"一部分行一部分不行"的真实形态)。

【截图:三人格合议·部分成立+评分+依据链】

### 3.3 规则沉淀门禁(人审反哺,"成熟一条应用一条")

人工采纳/驳回落库 → 自动生成规则草案进候选池 → **转正前在全部历史案卷自动回放**,产出"新增检出 X / 新增误报 Y / 漏检回退 Z"diff 报告 → 新增误报=0 且漏检回退=0 才放行,否则 409 退回候选池。假阳性率高的规则自动转 shadow 观察期。

【截图:回放门禁 diff 报告】

### 3.4 结构化输出与降级协议(生产化思维)

七环节 Agent 全部过统一包装器:schema 校验 → 带报错重试(≤2) → 按环节降级(分析失败→纯确定性结果;裁定失败→自动转人工)。每次重试/降级入台账(`/api/ops/degrade-log`),运维面板可查——兜底内嵌进系统,界面无感,内部有痕。

## 四、技术实现

### 4.1 分层架构

```
数据采集传输:  40表适配层(结算费用明细主锚) + PDF乱包通道(PP-Structure OCR + LLM字段抽取)
数据比对:      案卷对象编译(每条事实带源锚点) + 证据链完整度
违规筛查:      L1 确定性规则引擎(61条,毫秒级) → 明确违规档
明细审核:      L2 LLM 多Agent(读病历自由文本) + CoVe取证自检 → 可疑档
调查核实:      对抗合议三人格(立论→质证→裁定)
违规处理:      三要素证据链 + 处置建议(按违规性质分流) + 第35条算钱
评估分析:      领导版报告 + 机构画像 + 规则沉淀门禁
```

### 4.2 四级降级梯子(演示韧性=生产可用性)

LLM 调用 → ①超时/校验失败带错重试(≤2) → ②取预载真实结果 → ③纯确定性路径 → ④静态兜底视图。任何一级切换界面无感,全程记台账。

### 4.3 DRG 分组与算钱

分组器双形态:vendor OpenDRG(开源 CHS-DRG 分组器)在库即全量分组;演示子集为透明数据表(界面如实标注版本来源)。损失=高套组与应入组支付标准差额(实施细则第 35 条),公式随疑点卡展示。

### 4.4 模型与评测(真实数字,含没修完的)

- 目标模型:赛场内网千问;外网基线:SiliconFlow Qwen2.5-72B(辩手)/32B(异源裁判),2026-07-02 全量真跑(N=5/低温,零模拟,调用缓存可复现)
- **裁判环节(P5 v7)@ Qwen:两个裁判 verdict 6/6 全对、位置交换一致性 100%**(对照 MiniMax 裁判逐 rep 一致性仅 0.72)——裁判是"换模型家族收益最大"的环节
- 全量回归(82 个 用例×模型 组合,全绿=5/5 重复全过):v6 基线 **58/82 → v7 61/82 → v7.2 终态 68/82**。逐环节:P1 事实抽取 20/26→**26/26 满分**(修复提示词占位符病灶);P4 辩方认账 3/8→**7/8**(封闭式路径预检+金额算术清单);P2 规则判定 9/18→10/18;P3 8/8;P6 6/8;P7 11/14
- 已知短板如实呈现(归"模型能力/待内网复测"类,非 prompt 可修,均有下游兜底):①P2 时序判断与保守漂移(72B 特有,32B 多数通过)——生产侧 L1 时序规则(F-003 类)确定性兜底;②P4-R2 72B 铁证时偶发无据辩护(1/5)——辩方多辩不放走违规,P5 裁判(6/6 全对)+CoVe 是下游闸门;③P7-C1 归因——单次不达阈值本就"仅观察"
- 进场首小时跑 10 哨兵用例验证内网版本行为一致性;完整表格:eval/results/report_tables_qwen_*.md
