# 鹰眼 · 黄金链路（GOLDEN PATH）

> **这是「金剪刀」后的唯一主演示路径。** 宏大叙事保留在全面平台与 SSOT；本文只定义一条 10 分钟内可现场走完、每步有产物、结尾有门禁的闭环。
> 任何新功能合入前先回答：**「它有没有让黄金链路更硬？」** 没有 → 进 ROADMAP，不进演示。
> 页面入口：`/golden.html`（首页三入口之一）。

---

## Pitch 三入口结构

| 入口 | 页面 | 讲什么 | 时长 |
|---|---|---|---|
| **① 黄金闭环**（主演示） | `/golden.html` | 一个案卷走完「稽核→举证→复核→治理→复跑→测算→门禁」7 步闭环 | ~10 min |
| **② 全面平台**（纵深备问） | `/home.html` → 双工作台 + 看板 | 监管飞检 / 院端自查 / 优先队列 / Agent 编排 / 材料导入 / 工程看板——评委追问哪块点哪块 | 按需 |
| **③ 插件 & 小工具箱**（零对接嵌入） | `/plugins.html` → `/regulator-triage.html` 等 | 院端：老 HIS 开单/编码/结算事前提醒；监管：疑点分诊、阈值试算、飞检行装包，嵌进 Excel / 线索台账 / 飞检文件夹 | ~2 min |

三个入口共用**同一个引擎、同一套 KB、同一条 G0 红线**——③ 里的小工具不是另起炉灶，只是把主平台能力换输入方式和输出模板。

## 黄金链路七步（每步：动作 → API → 产物 → 口播）

| 步 | 动作 | API | 产物 | 口播要点 |
|---|---|---|---|---|
| 1 | **体检稽核** | `POST /api/audit?mode=exam` `{caseId:'main'}` | 报告（毫秒级）+ 自动落自查快照 | 40 分钟人工翻阅 → 秒级初筛；确定性引擎，可复现 |
| 2 | **三要素证据链** | （读第 1 步报告） | 每条疑点 = 证据定位 + 政策原文 + 推理链 | 引不出原文的疑点不输出；两库 ref 可点开 |
| 3 | **人工复核** | `POST /api/review` `{finding_id, rule_id, action, reason}` | 采纳/驳回落库留痕 | AI 不做最终认定，人是闸门 |
| 4 | **治理沉淀** | `GET /api/review` | 驳回≥3 自动标记高误报 → shadow | 误报不是修 bug，是有状态的治理 |
| 5 | **整改复跑 diff** | 再跑第 1 步 → `GET /api/exam/diff?case_id=main` | 已消除 / 仍存在 / 新增 三栏对比 | 自查→整改→再自查，整改效果可量化 |
| 6 | **主动退回测算** | `GET /api/exam/refund-estimate?case_id=main` | 按 38/40 条分档：建议退回 vs 飞检暴露区间 | 院端 ROI：自查从宽能省多少，一张表 |
| 7 | **门禁验收** | `GET /api/yhf` | G0 零误报 / G1 shadow / G2 prompt 门禁灯 | 每次改动都过这道门；G0 是不可破红线 |

**Definition of Done**：`/golden.html` 上 7 步全部点绿一遍 ≤ 10 分钟；`bash yhf/run.sh --strict` 全绿。

---

## 收敛纪律

- **黄金链路只用 `main` 案卷（NSCLC）+ exam 模式**。其余案卷/专科是全面平台的纵深，不进主演示。
- **KB 口径**：不讲「库里有 4583 条」，讲「本案每条疑点都能点到官方原文」。两库大盘只作检索背景，不作认定依据。
- **规则口径**：主演示只讲 demo 真 fire 的规则；58 条全量放合规地基面板供追问。
- **LLM 口径**：黄金链路默认确定性引擎（稳、可复现）；「真·LLM 对抗辩论」是全面平台里的增强按钮，不排进 10 分钟。
