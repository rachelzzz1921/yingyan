# 鹰眼 · 医保基金稽核规则库雏形 v0.1

> 用途：黑客松原型的稽核规则基线，供 Agent 循环推理调用与 RAG 知识库（KB1）挂接。
> 状态：草案，待确认 demo 场景后细化专科规则。
> 编制日期：2026-06-12 ｜ 法规基准：《医疗保障基金使用监督管理条例》（国务院令第735号）+《医疗保障基金使用监督管理条例实施细则》（国家医保局令第7号，2026-04-01施行）

---

## 一、设计原则（与产品承诺逐条对齐）

1. **证据链强制（三要素门禁）**：每条规则定义"最小证据集"。Agent 在输出任何疑点前自检三要素——① 原始证据定位（具体到费用行号、病历页码/段落原文）② 违反的政策条款原文（KB1 引用 ID + 原文）③ 完整推理过程。任一缺失，该疑点降级为"线索"或不输出。**宁漏报不误报**。
2. **三态输出**：
   - `疑点`（证据闭环）：三要素齐备，结论可直接用于与医疗机构对质；
   - `线索`（证据不全）：模式异常但材料包内无法闭环（如缺护理执行单），输出时必须附"需调阅材料清单"；
   - `不输出`：判断依赖未经证实的假设、或无法定位到任何原文。
3. **分层执行（对齐国家"两库"运行逻辑）**：国家两库框架中，违反"政策限定类"规则的结果是明确违规、可自动拦截；违反"合理使用类"规则的结果多为可疑、需听取医疗机构申诉。鹰眼对应分三层：
   - **L1 确定性规则**：字段与逻辑校验，纯代码可实现（性别、时间、数量、互斥），是底线，也是 LLM 输出的校验锚点；
   - **L2 语义规则**：必须读懂病历自由文本才能判定（本项目核心创新，超越传统规则引擎的字段比对）；
   - **L3 线索规则**：单份材料包不足以定论，需跨患者/跨病历批量数据或外部数据（执业库、进销存）佐证，只出线索。
4. **术语与官方对齐**：违规类型命名与《条例》第38/40条及国家医保局《定点医疗机构违法违规使用医保基金典型问题清单》保持一致，稽核报告可直接落到条款。
5. **参数化与本地化**：所有阈值（分解住院间隔天数、出院带药天数、护理时长上限等）做成可配置参数；地方价格目录、地方化问题清单以"覆盖层"挂接，不写死。

## 三、规则条目 Schema（机读定义）

```yaml
rule_id: string          # 唯一编号，如 B-201
rule_name: string
category: enum           # A收费合规 / B诊疗合理 / C住院行为 / D支付方式 / E资质数据 / F基础逻辑
violation_type: string   # 官方术语：重复收费 / 串换药品 / 虚构医药服务项目 ...
layer: enum              # L1确定性 / L2语义 / L3线索
risk_level: enum         # 高 / 中 / 低（初始定级；输出时结合证据完整度二次定级）
applies_to: [住院, 门诊]
specialty_tags: []       # 通用 / 骨科 / 心血管内科 / 血液净化 / 肿瘤 ...（对齐问题清单12领域）
trigger_logic: text      # 写给 Agent 的判定指令（自然语言 + 伪代码）
evidence_min_set: []     # 最小证据集；缺一不可输出为"疑点"
exclusions: text         # 除外情形 / 误报防控（宁漏报不误报的关键字段）
policy_basis: []         # KB1 引用 ID 列表
kb2_refs: []             # 临床指南引用（B/C 类规则需要）
params: {}               # 可配置参数（阈值、版本、地区覆盖）
demo_seed: text          # 模拟病历包预埋违规点建议
```

## 四、核心规则集 v0.1（33 条）

### F 基础逻辑类（L1 · 6条）——底线规则，全部可代码实现

**F-001 性别—项目冲突**（高｜虚记/串换线索）
- 触发：患者性别与性别限定项目冲突（男性出现宫腔镜、产科类；女性出现前列腺类）。
- 最小证据：费用行 + 病案首页性别。
- 除外：性别登记错误（需同时输出"核对身份信息"提示）。
- demo预埋：男性患者费用清单含"宫颈刮片"。

**F-002 年龄—项目冲突**（高）
- 触发：年龄与项目适用人群冲突（成人收新生儿护理；儿童收老年综合评估）。
- 最小证据：费用行 + 出生日期。

**F-003 时间逻辑冲突**（高｜虚构服务项目候选）
- 触发：费用发生日期晚于出院/死亡日期；手术收费日期早于入院；麻醉时间与手术记录时间矛盾。
- 最小证据：费用行日期 + 病案首页出入院（死亡）时间 + 相关记录单原文。
- demo预埋：出院日次日仍产生"二级护理"与输液费（飞检通报中反复出现的经典问题）。

**F-004 单日计量超物理上限**（高）
- 触发：单日累计数量超物理/规则上限：吸氧 >24小时/日、护理 >24小时/日、同一时刻两台全麻手术等。
- 参数：`max_per_day` 按项目配置。
- demo预埋：单日"持续吸氧"计费 30 小时。

**F-005 互斥项目同时计费**（中—高）
- 触发：互斥对照表命中（如同一静脉通道"静脉输液"与"静脉注射"同时段并收；全麻与局麻同台并收）。
- 除外：病历记载确有两次独立操作的，不触发——除外判断升级为 L2 由 Agent 读记录确认。

**F-006 限频项目超频次**（中）
- 触发：限"每日1次/每住院1次"项目重复计费（如入院诊查费、病房空气消毒按床日类项目）。
- 参数：限频表（从地方价格项目规范导入）。

### A 收费合规类（10条）

**A-101 重复收费——项目内涵已包含**（L2｜高｜重复收费→38条(三)）
- 触发：费用清单同时出现项目 X 与 Y，且按《医疗服务价格项目规范/立项指南》Y 属于 X 的"项目内涵"且不属"除外内容"。Agent 需读取 KB1 中 X 的内涵条文并比对。
- 最小证据：两条费用行 + 项目内涵条文原文 + （如涉操作）手术/操作记录原文。
- 除外：内涵条文标注"除外内容"可另收的；同日两次独立操作有记录支撑的。
- demo预埋：收"经皮椎体成形术"同时另收"椎体穿刺术"；收"心电监护"同日另收"血氧饱和度监测"。

**A-102 重复收费——同义/包含项目并收**（L2｜高）
- 触发：同一服务以不同名称/编码重复计费（"特级护理"与"重症监护"同日并收；同一检验项目以单项+组套双收）。
- 最小证据：费用行对 + 项目对照依据（KB1 同义映射表）。

**A-103 超标准收费——单价超物价标准**（L1｜高｜超标准收费）
- 触发：结算单价 > 当地医疗服务价格项目目录公示价。
- 依赖：地方价格库（KB1-L2层）。demo 若不锚定具体省份，此条仅做演示桩。

**A-104 超标准收费——扩大计价单位/数量**（L2｜高）
- 触发：按"次"计价项目按部位/小时叠加；计价单位与记录不符（如理疗一次治疗拆多部位计多次）。
- 最小证据：费用行 + 价格项目计价单位条文 + 治疗记录原文。

**A-105 超标准收费——护理等级与实际不符**（L2｜高）
- 触发：收费护理等级 > 医嘱护理等级，或护理记录频次不满足所收等级的巡视/记录要求。
- 最小证据：护理费用行 + 护理医嘱原文 + 护理记录单。
- demo预埋：医嘱"二级护理"，全程按"一级护理"计费。

**A-106 分解项目收费**（L2｜高｜分解项目收费）
- 触发：将打包项目拆分为多个子项分别计费（手术含的常规缝合、冲洗拆开收；组套检验拆单项叠加且合计高于组套价）。
- 最小证据：费用行组 + 打包项目内涵条文 + 价格对比计算过程。

**A-107 串换药品/耗材/诊疗项目**（L2｜高｜串换→38条(四)）
- 触发：实际实施/使用（以病历、医嘱、手术记录为准）与医保结算名目不一致：医嘱用 A 药结算 B 药；实施目录外项目（美容、保健类）按目录内治疗项目结算；低值耗材按高值结算。
- 最小证据：结算费用行 + 病历/医嘱中实际项目原文 + 两者目录属性对照（甲乙类/自费）。
- 除外：同通用名不同厂牌替代且价格一致的，提示而不定性。
- demo预埋：病程记录"行刮痧治疗"，费用清单结算"中医定向透药疗法"。

**A-108 虚记费用——收费项目无医嘱及执行记录支撑** ★三方验证核心（L2｜高｜虚构医药服务项目→40条(三)）
- 触发：对每条费用行（自动计费类除外），在长期/临时医嘱单、护理执行/治疗单、手术麻醉记录、检查检验报告中检索对应的开立或执行证据（名称需经同义归一：收费名↔医嘱名↔套餐名）；全部材料均无对应 → 疑点。
- 最小证据：费用行 + "已核对材料范围说明"（列明核对了哪些单据、关键阴性结论可复核——阴性证据也要可审计）。
- 除外：床位费等按床日自动计费项目；医嘱以套餐开立、费用按明细展开的（先做套餐展开再比对）；材料包缺整类单据（如未提供护理执行单）时降级为`线索`并列出调阅清单。
- demo预埋：收"伤口换药（大）"×6，医嘱与护理记录仅 2 次；收"雾化吸入"全程无医嘱。

**A-109 多记费用——结算数量超医嘱/记录数量** ★（L2｜高｜超量计费/多记）
- 触发：`结算数量 > 医嘱剂量×执行天数` 或 > 执行记录次数（允许损耗参数）。
- 最小证据：费用行数量 + 医嘱原文（剂量、频次、起止）+ 计算过程。
- demo预埋：医嘱"头孢呋辛 1.5g bid ×3天"（应为6支），结算 12 支。

**A-110 范围外费用纳入医保结算**（L2｜高｜38条(六)）
- 触发：医保结算清单中出现目录外项目/药品（含商保创新药目录品种——该目录品种医保基金不予支付）、生活服务类费用（陪护、保健品等），且结算标志为医保支付。
- 最小证据：费用行（含结算类别）+ 目录检索结论（KB1 目录行级引用）。

### B 诊疗合理类（L2 · 8条）——需要 KB2 临床指南库配合

**B-201 药品使用超出医保目录限定支付范围** ★演示效果最佳（L2｜高｜38条(六)）
- 触发：① 提取医保结算药品 → ② 查 KB1 目录"备注"列限定支付条件（"限"字条款）→ ③ 把限定条件解析为可核验要素（诊断要求/检验阈值/患者状态如重症抢救/给药前提如不能口服/治疗线数/基因检测要求/年龄/机构等级）→ ④ 在入院记录、病程、检验报告、医嘱中逐要素找证据 → ⑤ 任一必要要素存在明确反向证据或全材料无支持 → 疑点。
- 最小证据：费用行 + 目录备注原文（行级引用ID）+ 病历关键证据/阴性结论定位。
- 除外：患者自费结算未走医保的；限定条件语义模糊（如"重症"无量化口径）且病历无明确反向证据的，降级为线索；地方有放宽/收紧细则的以参数覆盖。
- demo预埋：① 人血白蛋白——病历白蛋白 38g/L、非重症非抢救，医保结算多支；② 某靶向药（备注限基因检测阳性）——全材料无基因检测报告。

**B-202 无指征使用抗菌药物**（L2｜中—高｜过度诊疗/不合理用药→38条(二)）
- 触发：使用治疗性抗菌药物，但病历无感染诊断、无感染证据（体温、血象、PCT/CRP、培养、影像），亦非围手术期预防适应情形。
- 最小证据：药品费用行/医嘱 + 病历检索结论（列明已核对的化验与病程记录）。
- kb2_refs：《抗菌药物临床应用指导原则（2015年版）》对应章节。
- 除外：预防用药符合切口类别与时机的；经验性用药且病程有疑似感染记载的（降级线索）。

**B-203 重点监控药品无适应症使用**（L2｜中）
- 触发：国家/省重点监控合理用药目录品种，病历诊断与该药说明书适应症无对应。
- kb2_refs：重点监控目录（第一批2019、第二批2023）+ 药品说明书。

**B-204 超说明书用药（适应症/用法用量）**（L2｜中）
- 触发：给药剂量、频次、途径或适应症超出说明书，且病历无超说明书用药审批/知情记录。
- 除外：有药事会备案或循证依据记载的，降级提示。

**B-205 重复用药——同类药理叠加**（L2｜中｜重复开药）
- 触发：同一时段开立两种同药理亚类、作用重叠药品（如两种质子泵抑制剂、两种同类中药注射剂）且无换药/合用理由记载。

**B-206 超量开药 / 出院带药超天数**（L1+L2｜中｜超量开药）
- 触发：出院带药 > 参数天数（急性病≈7日、慢性病≈14日、特殊慢病≈30日，按统筹区配置）；门诊单处方超量。
- 最小证据：出院带药医嘱/费用行 + 诊断类别判断依据。

**B-207 无指征检查（大套餐式检查）**（L2｜中—低｜过度检查）
- 触发：检查/检验项目与本次诊断及病情记录无关联（如单纯阑尾炎住院做全套肿瘤标志物+甲功），且病程无开单理由。
- 输出：默认仅出`线索`（医学合理性争议大，对齐"合理使用类规则结果为可疑"的官方口径），除非命中地方负面清单明确情形。

**B-208 康复/理疗超疗程且无评估**（L2｜中）
- 触发：康复治疗项目连续超过参数疗程仍计费，期间无阶段性康复评定记录。
- kb2_refs：康复医学诊疗规范；地方康复类项目限制条款。

### C 住院行为类（4条）

**C-301 分解住院**（L1+L2｜高｜38条(一)）
- 触发：同一患者前次出院至本次入院间隔 ≤ `N`日（默认15，按统筹区7–31配置）且主诊断相同/高度相关；并存在任一加重情节：前次出院记录"病情未愈/建议近期再入院"、两次治疗方案连续、前次费用临近支付标准上限。本次入院无新发病情的医学说明 → 疑点。
- 最小证据：两次住院的入出院记录+出院小结原文 + 间隔计算 + 诊断对照。
- 除外：放化疗周期、血透等按疗程规律住院；急诊新发；转诊转院。
- demo预埋：腰椎间盘突出第10天"好转出院"，第3日同诊断再入院延续同方案。

**C-302 挂床住院**（L2｜高｜38条(一)）
- 触发：住院期间连续 ≥ `M`日（默认3）无任何医嘱执行、护理记录、生命体征记录，或病程明示患者离院，但床位费/护理费/诊查费持续计费。
- 最小证据：费用行（连续计费日期）+ 体温单/护理记录空窗原文定位。
- 除外：请假外出有规范审批记录且对应日期未计费的。

**C-303 低标准入院（含体检式住院）**（L2｜中—高）
- 触发：入院记录不满足入院指征（症状体征轻、门诊可治），且住院期间医嘱以检查为主、无实质治疗。
- 最小证据：入院记录原文 + 全程医嘱结构统计（检查类占比）+ KB2 对应病种入院指征条目。
- 输出：证据闭环出疑点，否则线索（指征判断属医疗合理类，从严把握）。

**C-304 虚假住院（整册造假）**（L3｜高｜40条）
- 触发：单份材料内出现强信号（病程逐日雷同复制、检验值跨日完全一致、签名笔迹打印化等）；本质需跨病历比对。
- 输出：仅出线索 + 建议调取同期批量病历。

### D 支付方式类（DRG/DIP · 3条）

**D-401 高套分组——主诊断/主手术与病历不符**（L2｜高）
- 触发：病案首页主诊断/主手术编码在病历中无支持证据，或违反 DRG/DIP 2.0 编码校验规则（如使用"不作为入组条件"的诊断/手术作主要编码、把附加说明类编码升格为主诊断），导致入组权重升高。
- 最小证据：病案首页编码 + 病历反向证据（如编码"脓毒症"但无相应感染指标与病程支持）+ 2.0 分组规则条目引用。
- demo预埋：社区获得性肺炎编码为"重症肺炎"，但血气、氧合指数与病程不支持。

**D-402 费用转嫁——应保项目让患者自费/院外购买**（L2｜中—高）
- 触发：住院期间病历/医嘱出现"自备药/外购"且该药品目录内、属本次治疗必需；或目录内诊疗项目转为全自费结算无患者知情记录。
- 最小证据：医嘱或病程原文 + 目录属性 + 费用清单结算类别。

**D-403 服务不足/未达必要诊疗**（L3｜低）
- 触发：未完成临床路径关键环节即出院、住院日显著低于病组均值且结局记录缺失。仅出线索。

### E 资质与数据类（2条）

**E-501 无资质/超执业范围执业**（L3｜高）
- 触发：手术记录术者、麻醉记录签名人员与执业注册信息不符。依赖外部执业库，黑客松可做演示桩。

**E-502 病历资料矛盾/涂改痕迹（伪造变造线索）**（L2-L3｜高｜40条(二)）
- 触发：同一事实跨单据矛盾（手术记录时间 vs 麻醉单 vs 收费时间三方不一致）、扫描件可见涂改。
- 输出：线索 + 列明矛盾点对照表。

## 七、专科扩展目录（占位 · 对齐自查自纠 12 领域）

> 下列为各领域高发问题方向的占位规则，**待抓取官方 2024/2025/2026 年问题清单原文后逐条对齐措辞与编号**。

- 骨科：手术费已含一次性耗材另行收费；椎体成形/内固定术中拆分收取穿刺、复位类项目；植入耗材数量与手术记录不符。
- 心血管内科：介入术中重复收取造影/穿刺插管类内涵项目；心电监护与单项生命体征监测并收；起搏器程控超频次。
- 血液净化：透析当日重复收取置管护理/冲管；透析监测与透析费内涵重复；透析器复用违规计费。
- 康复：同一时段叠加多项治疗；超每日治疗单元上限；无康复评定持续治疗。
- 医学影像：三维重建等后处理违规加收；同部位短期重复检查无指征；报告费重复。
- 临床检验：组套拆分计费；无医嘱加做项目；超频次复查。
- 肿瘤：靶向药无基因检测证据；超适应症/超周期化疗；辅助用药滥用。
- 麻醉：麻醉时长与手术记录矛盾；麻醉中监测重复收费；术后镇痛无医嘱计费。
- 重症医学：特级护理与监护类项目内涵重复；呼吸机使用时长虚计；ICU 打包项目拆分。
- 口腔（2026新增领域）：种植/正畸等自费项目串换为医保项目结算；治疗次数虚计。
- 内分泌（2026新增领域）：动态血糖监测超频次；胰岛素泵耗材多记。
- 精神医学（2026新增领域）：心理治疗类项目无记录/超频次；长期住院患者实际离院仍计费。
