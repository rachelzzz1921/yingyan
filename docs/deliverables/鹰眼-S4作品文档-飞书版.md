## 00 · 侧重点声明：我们做哪一段

一句话把舞台灯光打准：**让一名基金监管科审核员，用一个工作台处理完今天的疑点队列，并让每一条结论都能回答"依据什么、怎么处置、规则如何进化"。**

医保基金监管是"事前提醒 → 事中审核 → 事后监管"的三阶段闭环。国家智能监管子系统已经把**结构化结算数据的大规模筛查**做到了毫秒级——谁先被查、哪笔直接拒付，这一段国家做得很好，我们不碰、也不重复建设。

鹰眼做的是它之后的那一段：**当一份由病历、病程、医嘱、检验、费用清单拼成的非结构化案卷摆在桌上，怎么在 90 秒内读懂它、查清它、并把结论写成经得起申诉复查的证据。** 这一段目前靠人逐份深读，是全链条上最耗人力、最无人填的空白——官方《医疗保障基金监督检查五年行动计划（2026—2030）》也把"AI+病例判读（提供线索支持）"列为探索方向，我们的定位与官方口径同构：AI 提供线索与举证支持，定性与处罚永远由人做。

## 01 · 项目简介：解决什么问题

**问题可以浓缩成一个人的一天。** 基金监管科审核员老李，每月面对智能监控筛出的一万多条疑似违规数据，初审、申诉后复审、再重审要过三遍；轮到飞检进场，一份案卷要把费用、病程、医嘱逐一对上，资深专家人均要读 40 分钟。他不缺线索——国家系统给的线索比他看得完的多得多——他缺的是**把线索变成证据的时间**。

这不是个案，是被官方数据坐实的结构性瓶颈：

| 硬数字 | 含义 | 出处 |
|---|---|---|
| 8,600 人 | 全国医保监管员总数，人均监管量约为美国 4 倍 | 官方统计·公开报道 |
| 13.27 亿 / 28.99 万家 | 参保人 / 定点医药机构（2024 统计公报） | 国家医保局统计公报 |
| 500 家 / 年 | 飞检查得过来的机构数，覆盖率约 0.17% | 2024 年飞检工作方案 |
| 40 分钟 | 单份案卷人工初筛耗时 | 一线稽核实践 |
| 342 亿元 | 2025 年追回基金总额——靠人海 + 抽查换来的 | 官方年度总结 |

**招人补不上这个缺口**（县级专职监管机构占比仅 3%），所以答案只能是让每个人顶更多人用。

**鹰眼 = 人少事多场景下的 AI 人力倍增器。** 输入一份案卷（支持 JSON / PDF / 扫描件 / 表格），引擎做"费用—诊断—医嘱"三方语义交叉验证，输出三档分诊（明确违规 / 可疑 / 干净——对齐国家两库"政策限定类 / 合理使用类"的官方二分，不自造概念），每条疑点带完整的**三要素证据链**：

1. **证据定位**——费用第几行、病历哪一页，点击回链原文
2. **政策条款**——两库哪一批、什么文号第几条、何时生效，点击可溯源
3. **推理过程**——为什么判、置信度多少、能不能扛住申诉

引擎实测：主演示案卷（肺癌化疗住院，虚构脱敏数据）**27 毫秒**完成 79 条规则路由（激活 29 条、50 条零成本跳过），产出 6 条疑点（涉及 ¥6,326）+ 4 条待核线索，12 处政策引用全部可解析到原文。人工 40 分钟的活，初筛这一步收敛到秒级——**人从大海捞针里解放出来，只花在真争议上。**

## 02 · 业务价值：对谁、值什么

同一台引擎，两侧落地——监管侧是价值标杆，院端是商业主场。

**对监管侧（医保局 / 飞检组 / 经办审核），鹰眼替掉六件最耗人的活：**

| 用处 | 替掉的人力活 | 产品支撑 |
|---|---|---|
| 飞检案卷取证 | 资深专家逐本翻病案找证据（单案 40 分钟） | 90 秒三要素证据链 + 一键举证包 |
| 优先排序 | 上万案卷不知先查谁 | 优先队列按金额 × 置信度排序，先查高风险 |
| 批量初筛 | 1 万多条疑点过三遍 | 1,000 条结算明细 19 毫秒三档分诊（实测命中 32 行，与预埋真值对账 32/32、误报 0） |
| 误报前置过滤 | 疑点越多误报越高，逐条人工排雷（同业案例：7,145 条疑点近四成金额为误报） | "引不出原文不输出" + 申诉预演，进入人工的都是"扛过一轮申诉"的 |
| 可审计的覆盖度 | 说不清"查了什么、有没有漏" | 每次输出覆盖度声明：覆盖了哪些维度、未触发是因本案无相关项而非漏查 |
| 现场小工具 | 医院不给导出数据、Excel 疑点清单靠手筛 | 零对接工具箱：屏幕 OCR 录入、Excel 分诊、规则阈值试算、飞检行装包 |

**对院端（医院医保办 / 医务科），鹰眼是政策强制动作的执行工具。** 国家自查自纠 2026 年扩至 12 个重点领域、回溯两年，官方明文"自查从宽、被查从严"。但真实差距是：某三甲医院飞检查出 82 项违规、涉及 3,466.7 万元，而它此前自查仅退回 3.23 万元——**千倍缺口，不是医院不想查，是手里的工具只会查结构化字段，语义自查层完全空白。** 鹰眼的院端模式（自查体检口径）给的是：按 12 领域清单批量体检历史案卷 → 带条款引用的整改清单 → 整改前后复跑对比 → 主动退回金额测算（主案卷实测：建议主动退回 ¥6,326，对比被飞检查实的暴露区间 ¥12,652～18,978——**自查从宽能省下的钱，一张表算给院长看**）。

**两侧是一根链条，不是两个产品**：医生开单那一刻被插件拦下的每一单，都是稽核员案头少掉的一条疑点；拦不住、对方坚持提交的，带着理由自动流入监管预警台账——**院端故事的终点，正是监管故事的起点**。这正是官方"关口前移、源头治本"的落地形态。

## 03 · AI 创新性：AI 在哪里、为什么是核心

先诚实回答"AI 在哪里"：**鹰眼不是把案卷扔给大模型要答案的产品。** 确定性规则引擎负责判"像不像"（毫秒级、可复现、可解释），AI 用在三个字段规则做不到、且必须做对的地方——**读懂自由文本的语义判读、模拟对质的三人格合议、定稿前的取证自检**。而真正让 AI 结论能被监管场景接受的，是包在外面的三层工程约束。这三层是本作品的核心创新。

**第一层：取证可信度三件套——让"可信"成为数据结构，而不是口号。**

| 机制 | 做什么 | 替监管回答了哪个致命问题 |
|---|---|---|
| 合议去重 | 同一笔费用被多条规则从不同角度命中时，自动选主疑点、其余转佐证、金额只算一次，输出去重日志（主案卷实测：¥1,280 的费用行被 3 条规则命中，只计一次，避免虚增 ¥2,560） | "你会不会重复计罚？"——防过罚是稽核大忌 |
| 覆盖度声明 | 输出"本次覆盖 7 个维度中的 6 个，未触发维度是因本案无相关项而非漏查"（主案卷缺失的那 1 项——分子病理/基因检测证据——恰恰本身就是一条疑点的关键阴性证据） | "你怎么证明你没漏？"——把漏检从认怂变成可审计的签字声明 |
| 置信度传播 | OCR 低置信证据（<0.85）按比例降权；关键证据置信不足，疑点自动降级为线索 | "模型会不会幻觉乱报？"——反幻觉是可追溯的计算 |

这三样不是模型能力，是**要懂稽核实务（过罚、漏检率、申诉）才设计得出的流水线**——通用大模型和字段规则引擎都没有这一层。

**第二层：三人格合议 = 把法定申诉机制预演到初筛。** 真实世界里，事中退回可申诉、飞检判定后有申诉，医院基于药理医理申诉、医保专家团队评定（医保办发〔2023〕14 号规定的"复审、合议、终审"）。鹰眼把这套对质**前置**：可疑档启动控方（只看规则与结算数据）、辩方（只看临床材料）、裁判（必须引用知识库条目，引不出即转人工）的三方合议——**信息不对称是在数据层物理切分的，不是提示词自律**。主案卷的升白针线索就是活例：控方主张超限定支付，辩方举出高风险化疗方案的预防指征，裁判钢人复述双方最强论点后降级为线索待调阅——**误报在见到人之前就被过滤掉了，同时为院端自动生成了申诉举证材料**。我们如实切分：演示默认走预置推演脚本（系统真实运行的产出、预先载入），现场可切换真·大模型路径实时重跑。

**第三层：规则沉淀门禁——让系统进化也过审。** 审核员的每次采纳/驳回回流为候选规则；候选转正前必须在全部历史案卷上自动回放，产出"新增检出 / 新增误报"的 diff 报告，**新增误报不为零就不放行**；被高频驳回的在役规则自动转入观察期。对应官方"成熟一条、应用一条，假阳性率高的及时停用"的原话——政策给了地方增补规则、自设阈值的权限，但没给工具，**这就是那个工具**。

一句话收束：**规则引擎判"像不像"，AI 判"说得通说不通"，三件套保证"算得对、说得清、不乱报"。我们不是相信模型，是不信任模型，所以造了一条可审计的流水线。**

## 04 · 技术实现：整体思路

**六层流水线**，每一层只干一件事：

1. **摄取**——JSON / PDF / 扫描件 / 表格多格式导入，OCR 回填锚点坐标与置信度
2. **事实**——把材料包一次性编译为类型化"案卷对象"，每条事实自带源锚点；证据定位从"模型临场摘录"变成数据结构的硬字段
3. **路由**——触发器路由 + 三级短路，只激活与本案相关的规则（实测 79 条中激活 29 条，50 条零成本跳过——语义分析的开销按需分配，不是每条都烧一遍）
4. **判定**——确定性 checker 打底，语义判读按需增强；合议层去重、定档
5. **门禁**——三要素完整性（缺一降级）、三人格合议、取证自检、置信度校准
6. **治理**——规则三态（在役 / 观察 / 下线）、误报回流、沉淀回放门禁

**双路径的分工与两库同构**：确定性引擎对应两库"政策限定类"（明确违规，毫秒级自动拦截），大模型语义路径对应"合理使用类"（可疑，要听申诉）——技术分层不是我们发明的，是把官方分类工程化。

**知识库工程（数据地基，全部可点验）**：

- 官方两库及政策条目已入库 **11,027 条**（法规 56 · 问题清单 203 · 规则 10,340 · 目录 395 · 判例 29），来源以国家医保局公开发布批次为主，**7,959 条带生效日期**——支持按入院日期取"当时有效"的政策版本判当时的案卷，这是飞检追溯认定的核心难点
- 从两库 10,337 行规则条目提炼出 16 个模式族、7,822 项操作索引，其中 60% 已接确定性检测
- 手写操作化为在库规则 **79 条**（横跨 19 个临床专科），**79/79 全部挂接引擎 checker**（覆盖地图逐格可点）
- 规则的 118 处政策引用，**106 处（90%）可一键解析到官方原文**（文号 · 条款 · 生效日 · 核验状态）；11 项被引用但未入库的对照表诚实标注"待入库"——这是路线图，不是掩饰
- 我们**不造规则**：只接官方已公开发布批次（截至 2026-06 约 16 批），把国家下放给地方的"阈值、参数、应用场景"落到具体案卷上

**工程纪律（为什么敢说低误报）**：金标准、运行态、观察期三套数据物理隔离；任何规则/文案/模型变更都要过变更门禁——**干净案卷误报必须为零，这是一票否决的红线，不是优化目标**。评测走真实模型全量回归（Qwen2.5-72B/32B，零模拟）：82 用例从 58 提升到 68（多轮合成终态，单次最高 61）；裁判环节位置交换双跑一致性 100%；事实抽取 26/26。我们同样如实标注：基准案卷为团队构造自标、demo 级，真实泛化误报率需试点验证——**我们主打的不是"考了 100 分"，是"怎么防止自己骗自己"的机制**。

## 05 · 交互与设计：三个入口，三个故事

打开产品首页，只有三个入口——对应三个能独立讲完的故事，共用同一台引擎、同一套知识底座、同一条零误报红线。

**入口一 · 黄金闭环（10 分钟）："一份案卷是怎么查清楚的。"** 七步解锁递进：选材料（可现场拖入 PDF 乱包）→ 秒级初筛 → 三要素证据链逐条核对 → 人工采纳/驳回 → 复核意见沉淀 → 整改后复跑对比（已解决 / 仍存在 / 新出现三栏）→ 主动退回测算 → 质量门禁验收。页面带演示计时器实时走表，走完全程给出三个出口：换一份材料再走、带着案卷进完整平台、返回首页。

**入口二 · 完整平台（双前台）："每天一万多条，是怎么管过来的。"**

- 监管前台（稽核口径）：优先队列 1,000 条三档分诊 → 点一条进单案工作台 → 左栏费用行为锚、点行看"钱→行为→指征"证据链 → 右栏报告与举证包导出 → 采纳驳回反哺规则治理

- 溯源交互三层锚点：疑点卡 → 点证据定位跳转费用行/病历原文并高亮 → 点政策条款进知识库条目直至官方文件页码。**每条结论都点得开，是界面的第一设计原则**

## 06 · 真实性与可验证性：每个说法怎么当场验

这一章是为"这是不是录好的"这类怀疑准备的。本文所有关键数字都有当场验证入口：

| 你想验证什么 | 当场怎么验 |
|---|---|
| 疑点是真算的还是写死的 | 任指一条疑点，现场展示它由哪几个字段交叉得出；或现场改动案卷数据点"重新稽核"，毫秒级重算 |
| 数据是不是编的 | 数据结构取自官方四十张脱敏表，违规样例仿国家局曝光台真实案例构造，每条埋点有底账清单（ground-truth manifest），指哪条对哪条 |
| 批量筛查有没有误报 | 1,000 条筛查结果页自带"真值对账"：预埋 32 / 检出 32 / 误报 0，逐行可点 |
| 规则是不是自己编的 | 打开"合规地基"面板：每条规则溯到哪份官方文件、哪一条、哪天生效，逐条点开原文；打开覆盖地图看 79/79 挂接状态 |
| 合议是脚本还是真跑 | 默认预置推演（如实标注），现场可切真·大模型路径任选一条疑点实时对抗，约 80 秒出结果——慢一点，但是真的 |

## 08 · 商业与落地路径

**现金流主战场在院端，监管侧是信誉锚。** 院端自查是"政策强制 + 官方鼓励"出来的存量市场：全国万级定点机构每年必须自查、决策链短（医务科可先试用）、付费动力被政策拉满——一次飞检罚款百万到千万级，工具费是零头，ROI 锚定"避免一次飞检损失"而不是"提效"。行业侧面印证：院端 DRG/DIP 相关信息化投入约 141 亿元、年复合增速约 18.5%，但现有工具全部停在结构化/编码质控层——**院端不是没工具，是买了错误层的工具**，语义自查层留白。

- **近期**：院端"自查体检"轻服务切入（材料包导入、数据不出院、可离线私有化），按机构/科室年度订阅；监管侧按项目/坐席，走《飞行检查管理暂行办法》"聘请 IT 服务机构协助飞检"的法定通道
- **冷启动**：规则驱动 + 专家标注少量金标，不依赖大规模真实病历训练，合规友好；团队已有医院/医保局资源可承接种子试点
- **路径**：先在医院落地自查（现金流）→ 积累规则资产与机构画像 → 反哺监管侧标杆（信誉与天花板）
- **护城河**：业务 know-how + 规则资产积累（79 条操作化 + 万级官方条目入库是脏活累活，不是一晚能堆的 demo）+ 学术与专家背书——我们不讲"数据飞轮"：私有化部署下数据不跨院，真壁垒是把脏活做完的工程纵深

## 09 · 团队与工程纪律

**两人互补，从政策条款到可跑产品的闭环在团队内完成。** 队长陈知维：证券医疗行研出身，负责医保业务建模、产品设计与商业化——知道飞检现场怎么吵、医院怎么申诉、稽核员怎么写报告；队员赵馨：AI 与全栈工程，RAG 检索、模型部署到交互平台独立落地。北京大学年庆功博士全程专业指导。
