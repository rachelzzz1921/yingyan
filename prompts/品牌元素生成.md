## Prompt 正文（从这里复制）

```
你是一位 B2G 政务科技 + 医疗监管领域的品牌设计顾问。请为以下产品生成完整的品牌元素方案。

---

## 一、项目背景

**产品名**：鹰眼（英文可写 Yingyan / EagleEye Audit）

**全称**：医保基金稽核智能体

**一句话定位**：稽核员的 AI 初筛员——读懂非结构化病历，90 秒输出可对质的证据链稽核报告。

**Slogan（已有，可优化或出备选）**：让每一分救命钱，都查得有据

**核心用户**：医保局飞检稽核员、医保经办核查人员、医院医保办（自查模式）

**使用场景**：飞行检查现场、稽核作业台、Pitch 路演、黑客松 Demo、私有化部署控制台

**产品本质**：不是消费医疗 App，不是医院 HIS，不是通用 SaaS。是「站在国家官方规则库肩上的语义增强层」——国家系统筛线索，鹰眼做取证放大镜。

**品牌人格关键词**：
- 监管可信（严肃、可审计、有依据）
- 智能锐利（精准定位、不废话、有锋芒）
- 证据导向（每个结论可回链原文）
- 宁漏报不误报（克制、负责任）

**核心差异化（必须在品牌中体现）**：
1. 三要素证据链：原始证据定位 + 政策条款原文 + 完整推理过程
2. 引不出原文的疑点，不输出
3. 控辩裁多 Agent 对抗复核
4. 规则本身也要过三审三验

**竞品参照（我们要避开他们的视觉气质）**：
- 大厂医保信息化：蓝灰企业风、集成平台感
- 金豆/院端 DRG：运营工具感、偏医院端
- 消费医疗 App：高饱和、圆润、亲和

**我们要占的位置**：飞检台上的「取证放大镜」——介于政务监管系统与 AI 工具之间

---

## 二、已有品牌规范 v1.0（可在此基础上延展，也可提出 v2 变体）

**色彩 Token**：
| 名称 | 色值 | 用途 |
|---|---|---|
| yy-ink | #0B2A4A | 主文字、顶栏深底 |
| yy-ink-2 | #134E72 | 渐变辅色 |
| yy-muted | #5C7185 | 次要文字 |
| yy-iris | #2DD4BF | 鹰眼虹膜、成功强调 |
| yy-amber | #E8A838 | 扫描环、品牌点缀 |
| yy-danger | #DC4A3D | 疑点、红线 |
| yy-warn | #C77700 | 线索 |
| yy-ok | #0D9B6A | 合规、PASS |
| yy-bg | #F0F4F8 | 页面底 |
| yy-surface | #FFFFFF | 卡片 |

**图形语言（已有 logo-mark 概念）**：
- 抽象鹰眼 + 琥珀色扫描弧 + 十字准星（证据定位）
- 背景：subtle 网格点阵（监管/数据感）
- 顶栏：深蓝渐变（#061829 → #0B2A4A → #134E72）
- 卡片：白底 + 轻阴影 + 12px 圆角；疑点左缘色条

**字体**：
- 中文：Noto Sans SC
- 英文/数字：DM Sans（金额用 tabular nums）

**明确禁止**：
- 纯红医疗十字作为主视觉
- 过高饱和渐变（廉价 SaaS 感）
- 大面积紫色（purple 仅用于 shadow 观察期治理态，不是主品牌色）
- 卡通鹰、翅膀展开、美国国徽式 eagle
- 过于圆润的「友好医疗」风格

## 三、请生成的品牌元素（按模块输出）

### 模块 1 · 品牌命名与文案
- 中文产品名备选（3–5 个，说明取舍理由）
- 英文/拉丁名备选（适合 logo 与域名）
- Slogan 主标语（3 个方向，每个附 15 字以内副标语）
- 电梯版一句话（30 字内，给非技术评委）
- 品牌金句库（10 条，适合 PPT 大字报 / 海报）
- 品牌禁用词（5–8 个，如「智能诊断」「AI 医生」等）

### 模块 2 · Logo 与图形标识（文字描述 + 可给图像 AI 的 prompt）
请输出 **3 套差异化 logo 方向**，每套包含：
- 方向命名（如「准星取证型」「扫描环型」「文书透镜型」）
- 设计理念（50 字）
- logo-mark 构成元素拆解
- 横版 wordmark 排版建议
- 单色 / 反白 / favicon 16×16 适配说明
- **图像生成 prompt**（英文，适合 DALL·E / Midjourney，要求：矢量感、扁平、无文字、64×64 可识别）

### 模块 3 · 色彩系统延展
- 在现有 Token 基础上，提出 1 套「深色模式」色板
- 1 套「打印/PPT 安全」色板（CMYK 近似）
- 语义色使用规范（疑点/线索/合规/治理 shadow 四态）
- 渐变与阴影 token 建议

### 模块 4 · 字体与排版
- 标题 / 正文 / 数据 / 政策引用 四级排版规范
- 中英文混排规则
- PPT 与 Web 的字号阶梯
- 若不用 Noto Sans SC，给出 2 个备选字体及理由

### 模块 5 · 图形语言与 UI 纹理
- 背景纹理（点阵/网格/扫描线）3 种变体描述
- 图标风格指南（线宽、圆角、填充规则）
- 空状态 / 加载 / 扫描中 3 个插画场景描述
- 稽核报告 UI 中的「证据链卡片」视觉规范

### 模块 6 · 品牌语调（Voice & Tone）
- 对外 Pitch：正式、有数据、不夸大
- 产品 UI 微文案：简洁、可执行、无恐吓
- 错误/空状态：专业、不甩锅
- 各写 3 条示例句

### 模块 7 · 应用场景物料
- Pitch Deck 封面 3 种构图描述（含主标题位置、背景、点缀元素）
- 黑客松展台背板一句话 + 视觉主元素
- 名片 / 工牌信息层级
- 社交媒体头像与横幅尺寸建议

### 模块 8 · 品牌故事（可选，100 字）
- 「为什么叫鹰眼」的品牌叙事（连接监管场景，不要神话寓言腔）

---

## 四、输出格式要求

1. 用 Markdown 表格和分级标题，便于直接粘贴进设计文档
2. 每个模块先给「推荐方案」，再给「备选方案」
3. Logo 方向之间必须有明显视觉差异，不能只是换色
4. 所有方案必须能回答：**为什么稽核员看了会信任，而不是觉得又是一个 AI 噱头？**
5. 标注每个方案的「风险点」（如：太像某竞品、太小众、印刷不清等）

---

## 五、参考金句（可吸收、改写，勿照搬）

- 引不出原文的疑点，我们不输出
- 宁漏报，不误报——可信度是稽核场景唯一的硬通货
- 国家系统筛线索，鹰眼做取证放大镜
- 能不报，比能报更难，也更值钱
- 疑点有三要素门禁，规则进库要过三审三验

---

请开始生成。若信息不足，先列出不超过 3 个澄清问题，否则直接输出完整方案。
```
